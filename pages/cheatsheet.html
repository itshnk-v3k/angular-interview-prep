3. switchMap vs mergeMap vs concatMap vs exhaustMap

OperatorBehaviorTypical use
switchMapCancels the previous inner when a new value arrives.Autocomplete / type-ahead search, latest wins.
mergeMapRuns all inner Observables in parallel, no ordering.Independent parallel requests.
concatMapQueues inners, runs strictly one at a time in order.Sequential writes / ordered saves.
exhaustMapIgnores new values while an inner is running.Login button / double-submit protection.

4. Subject types

  • Subject — multicast, no initial value; subscribers receive only values emitted after they subscribe.
  • BehaviorSubject — requires an initial value, replays the latest value to each new subscriber.
  • ReplaySubject — replays the last N (or all, within a time window) values to new subscribers.
  • AsyncSubject — emits only the final value, and only after the source completes.

5. Lifecycle hooks (in order)

  1. ngOnChanges — an input changed.
  2. ngOnInit — initialization.
  3. ngDoCheck — custom change detection.
  4. ngAfterContentInit — projected content is ready.
  5. ngAfterContentChecked — projected content checked.
  6. ngAfterViewInit — view is ready.
  7. ngAfterViewChecked — view checked.
  8. ngOnDestroy — cleanup.

Modern alternatives: afterNextRender, afterRender (browser-only DOM work).

6. OnPush change detection triggers

  • An @Input() reference changes (new object/array, not a mutation).
  • A DOM event fires from the component or its template.
  • An async pipe (or signal) bound in the template emits a new value.
  • Change detection is triggered manually via ChangeDetectorRef.markForCheck().

7. Signals API

  • signal(v) — writable reactive value; read via s(), updated via set / update.
  • computed(fn) — derived, memoized, read-only signal, recomputed lazily.
  • effect(fn) — runs a side effect whenever any read signal changes.
  • toSignal(obs$) — converts an Observable into a signal (auto-unsubscribe).
  • toObservable(sig) — converts a signal into an Observable.
  • input() — signal-based component input (input.required<T>() for required).
  • model() — two-way bindable input signal ([(value)]).
  • linkedSignal() — writable signal that resets from a computed source.

8. DI provider types

ProviderMeaning
useClassInstantiates the given class for the token.

9. SOLID

  • S — Single Responsibility: a class/module should have one reason to change.
  • O — Open/Closed: open for extension, closed for modification.
  • L — Liskov Substitution: subtypes must be usable wherever the base type is expected.
  • I — Interface Segregation: prefer many small interfaces over one large one.
  • D — Dependency Inversion: depend on abstractions, not implementations.

10. Common interview questions

Q1. == vs ===? A: === compares value+type without coercion; == coerces first — always use ===.

Q2. null vs undefined? A: undefined = not assigned; null = intentionally empty value.

Q3. What is a closure? A: A function that retains access to its outer lexical scope even after that scope has returned.

Q4. let vs var? A: let is block-scoped with a TDZ; var is function-scoped and hoisted as undefined.

Q5. Microtask vs macrotask? A: Promises = microtasks (processed first); setTimeout = macrotasks (one per event-loop tick).

Q6. Arrow vs regular function? A: Arrows have no own this/arguments; they bind this lexically.

Q7. interface vs type? A: Interfaces merge and extend (objects); types do unions/intersections/primitives.

Q8. any vs unknown? A: any disables checks; unknown is safe — you must narrow before using it.

Q9. readonly vs const? A: const locks the binding; readonly locks a property at compile time.

Q10. Component vs Directive? A: A component has a template; a directive adds behavior to an existing element.

Q11. Standalone vs NgModule? A: Standalone components declare their own imports, no module needed (default in v17+).

Q12. Constructor vs ngOnInit? A: Constructor = DI setup; ngOnInit = initialization after inputs are set.

Q13. ViewChild vs ContentChild? A: ViewChild queries your own template; ContentChild queries projected content.

Q14. Template-driven vs reactive forms? A: Template-driven = directive-based in HTML; reactive = explicit model in TS (testable, scalable).

Q15. Subject vs BehaviorSubject? A: BehaviorSubject requires an initial value and replays the latest to new subscribers.

Q16. Cold vs hot Observable? A: Cold restarts for each subscriber (HTTP); hot is shared/multicast (Subject, events).

Q17. How to avoid memory leaks? A: Unsubscribe via takeUntilDestroyed, the async pipe, or take(1).

Q18. switchMap vs mergeMap? A: switchMap cancels the previous inner; mergeMap runs them all in parallel.

Q19. What is change detection? A: Angular checking bindings to sync the model with the DOM (Zone.js or signals).

Q20. Lazy loading? A: Loading route bundles on demand via loadComponent / loadChildren to shrink the initial bundle.

12. Router guard types

  • canActivate — allows/blocks navigation into a route.
  • canActivateChild — allows/blocks navigation into child routes.
  • canDeactivate — allows/blocks leaving a route (e.g. unsaved-changes warning).
  • canMatch — decides whether a route definition matches at all (replaces canLoad; controls lazy bundles).
  • resolve — pre-fetches data before the route activates.

13. Control flow in Angular 17+

  • @if (cond) { } @else if () { } @else { } — conditional rendering (replaces *ngIf).
  • @for (item of items; track item.id) { } — loop; track is required (replaces *ngFor).
  • @empty { } — fallback block shown when the @for collection is empty.
  • @switch (val) { @case (x) {} @default {} } — multi-way branching (replaces *ngSwitch).
  • @defer (on viewport) { } @placeholder { } @loading { } @error { } — lazy-load a template block.

15. OOP principles

  • Encapsulation — bundles state and behavior, hides internals behind a public API (#private).
  • Abstraction — exposes what an object does, hides how it does it.
  • Inheritance — derives a class from another to reuse and extend behavior (extends).
  • Polymorphism — one interface, many implementations; behavior resolved at call time.

4. Типы Subject

  • Subject — мультикаст, без начального значения; подписчики получают значения, эмитированные только после подписки.
  • BehaviorSubject — требует начальное значение, повторяет последнее значение каждому новому подписчику.
  • ReplaySubject — повторяет последние N (или все, в пределах временного окна) значений новым подписчикам.
  • AsyncSubject — эмитит только финальное значение и только после завершения источника.

5. Хуки жизненного цикла (по порядку)

  1. ngOnChanges — изменился input.
  2. ngOnInit — инициализация.
  3. ngDoCheck — кастомное обнаружение.
  4. ngAfterContentInit — проекция готова.
  5. ngAfterContentChecked — проекция проверена.
  6. ngAfterViewInit — представление готово.
  7. ngAfterViewChecked — представление проверено.
  8. ngOnDestroy — очистка.

Современные альтернативы: afterNextRender, afterRender (работа с DOM только в браузере).

6. Триггеры обнаружения изменений OnPush

  • Меняется ссылка @Input() (новый объект/массив, не мутация).
  • Срабатывает DOM-событие от компонента или его шаблона.
  • Пайп async (или сигнал), привязанный в шаблоне, эмитит новое значение.
  • Обнаружение изменений запускается вручную через ChangeDetectorRef.markForCheck().

7. Signals API

  • signal(v) — записываемое реактивное значение; читается через s(), обновляется через set / update.
  • computed(fn) — производный, мемоизированный, доступный только для чтения сигнал, пересчитывается лениво.
  • effect(fn) — выполняет побочный эффект при изменении любого читаемого сигнала.
  • toSignal(obs$) — преобразует Observable в сигнал (авто-отписка).
  • toObservable(sig) — преобразует сигнал в Observable.
  • input() — input компонента на основе сигналов (input.required<T>() для обязательного).
  • model() — двусторонне связываемый сигнал-input ([(value)]).
  • linkedSignal() — записываемый сигнал, сбрасывающийся из вычисляемого источника.

9. SOLID

  • S — Единственная ответственность: у класса/модуля должна быть одна причина для изменения.
  • O — Открытость/закрытость: открыт для расширения, закрыт для модификации.
  • L — Подстановка Лисков: подтипы должны быть применимы там, где ожидается базовый тип.
  • I — Разделение интерфейсов: лучше много мелких интерфейсов, чем один большой.
  • D — Инверсия зависимостей: зависьте от абстракций, а не от реализаций.

10. Частые вопросы на собеседовании

В1. == vs ===? О: === сравнивает значение+тип без приведения; == сначала приводит — всегда используйте ===.

В2. null vs undefined? О: undefined = не присвоено; null = намеренно пустое значение.

В3. Что такое замыкание? О: Функция, сохраняющая доступ к своей внешней лексической области даже после её завершения.

В4. let vs var? О: let имеет блочную область и TDZ; var имеет функциональную область и поднимается как undefined.

В5. Микрозадача vs макрозадача? О: Promise = микрозадачи (обрабатываются первыми); setTimeout = макрозадачи (по одной за тик цикла).

В6. Стрелочная vs обычная функция? О: У стрелочных нет собственного this/arguments; они связывают this лексически.

В7. interface vs type? О: Интерфейсы сливаются и расширяются (объекты); типы делают объединения/пересечения/примитивы.

В8. any vs unknown? О: any отключает проверки; unknown безопасен — нужно сузить перед использованием.

В9. readonly vs const? О: const фиксирует привязку; readonly фиксирует свойство на этапе компиляции.

В10. Компонент vs Директива? О: У компонента есть шаблон; директива добавляет поведение существующему элементу.

В11. Standalone vs NgModule? О: Standalone-компоненты объявляют свои imports, модуль не нужен (по умолчанию в v17+).

В12. Конструктор vs ngOnInit? О: Конструктор = настройка DI; ngOnInit = инициализация после установки input'ов.

В13. ViewChild vs ContentChild? О: ViewChild запрашивает собственный шаблон; ContentChild запрашивает проецируемый контент.

В14. Шаблонные vs реактивные формы? О: Шаблонные = на директивах в HTML; реактивные = явная модель в TS (тестируемые, масштабируемые).

В15. Subject vs BehaviorSubject? О: BehaviorSubject требует начальное значение и повторяет последнее новым подписчикам.

В16. Холодный vs горячий Observable? О: Холодный перезапускается для каждого подписчика (HTTP); горячий общий/мультикаст (Subject, события).

В17. Как избежать утечек памяти? О: Отписывайтесь через takeUntilDestroyed, пайп async или take(1).

В18. switchMap vs mergeMap? О: switchMap отменяет предыдущий внутренний; mergeMap запускает все параллельно.

В19. Что такое обнаружение изменений? О: Проверка Angular привязок для синхронизации модели с DOM (Zone.js или сигналы).

В20. Ленивая загрузка? О: Загрузка бандлов маршрутов по требованию через loadComponent / loadChildren для уменьшения начального бандла.

12. Типы guard'ов маршрутизатора

  • canActivate — разрешает/запрещает переход в маршрут.
  • canActivateChild — разрешает/запрещает переход в дочерние маршруты.

  • resolve — предзагрузка данных до активации маршрута.

15. Принципы ООП

  • Инкапсуляция — объединяет состояние и поведение, скрывает внутренности за публичным API (#private).
  • Абстракция — показывает, что объект делает, скрывает, как он это делает.
  • Наследование — порождает класс от другого для переиспользования и расширения поведения (extends).
  • Полиморфизм — один интерфейс, много реализаций; поведение определяется в момент вызова.
