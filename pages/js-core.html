Основы JavaScript

Фундамент языка, который на собеседованиях уровня senior проверяют особенно дотошно: типы, массивы и объекты, прототипы, замыкания, цикл событий, промисы и this.

Типы данных

В JavaScript 8 типов данных: 7 примитивов плюс object. Примитивы неизменяемы и сравниваются по значению; объекты изменяемы и сравниваются по ссылке.

number — число двойной точности (64-битное IEEE-754 с плавающей точкой). Охватывает целые и дробные, а также NaN, Infinity, -Infinity. Помните про 0.1 + 0.2 !== 0.3.

bigInt — целые числа произвольной точности с суффиксом n (9007199254740993n). Используйте, когда значения превышают Number.MAX_SAFE_INTEGER (например, ID заявок, сатоши).

string — неизменяемая последовательность кодовых единиц UTF-16. Любая «мутация» возвращает новую строку.

boolean — true / false.

undefined — переменная объявлена, но ей не присвоено значение. «Отсутствие значения» по умолчанию в JS.

null — намеренное, явное отсутствие какого-либо объектного значения. Вы присваиваете его осознанно.

object — единственный ссылочный тип: обычные объекты, массивы, функции, Date, Map, Set — всё, что не примитив.

symbol — уникальный неизменяемый идентификатор. Применяется для ключей объектов без коллизий и для well-known протокольных хуков (Symbol.iterator).

Классическая ловушка: typeof null === 'object'. Это историческая ошибка из самой первой реализации JS (тег типа для объектов был 0, а null был нулевым указателем 0x00). Исправить её, не сломав весь веб, невозможно, поэтому она остаётся.

typeof 42;            // "number"
typeof 9007199254740993n; // "bigint"
typeof "BTCUSD";      // "string"
typeof true;          // "boolean"
typeof undefined;     // "undefined"
typeof null;          // "object"  ← historical bug
typeof {};            // "object"
typeof Symbol('id');  // "symbol"
typeof function(){};  // "function" (a callable object)

Примитивы против ссылочных типов

Примитив — копируется по значению. При присваивании новой переменной значение дублируется; обе переменные независимы.

Ссылочный тип — копируется по указателю. При присваивании копируется ссылка, а не данные. Обе переменные указывают на один и тот же объект в памяти.

АспектПримитив (number, string…)Ссылочный (object, array)
Как хранитсяСамо значениеУказатель на память в куче
Семантика копированияПо значению (независимая копия)По ссылке (общая)
Сравнение=== сравнивает значение=== сравнивает идентичность
ИзменяемостьНеизменяемыйИзменяемый
 
// Primitive — copied by value
let a = 10;
let b = a;     // b gets its own copy
b = 20;
console.log(a); // 10 — unaffected

// Reference — copied by pointer (the mutation problem)
const order1 = { symbol: 'EURUSD', qty: 100 };
const order2 = order1;      // same object!
order2.qty = 500;
console.log(order1.qty);    // 500 — surprise, both changed

// Identity vs equality
console.log({} === {});     // false — different references
const r = {}; console.log(r === r); // true — same reference

Решения — как сделать настоящие копии

const order = { symbol: 'EURUSD', qty: 100, meta: { source: 'API' } };

// 1. Spread — SHALLOW copy (nested objects still shared)
const copy1 = { ...order };
copy1.meta.source = 'WS';   // ⚠️ also changes order.meta.source

// 2. Object.assign — also SHALLOW
const copy2 = Object.assign({}, order);

// 3. structuredClone — DEEP copy (modern, handles nested, Map, Set, Date)
const copy3 = structuredClone(order);
copy3.meta.source = 'WS';   // ✅ order.meta.source unchanged

Почему это важно в Angular: обнаружение изменений OnPush и редьюсеры NgRx опираются на новые ссылки. Spread достаточно для поверхностных обновлений; используйте structuredClone, когда нужно скопировать вложенное состояние.

Методы массивов

Понимание того, какие методы мутируют массив, а какие возвращают новый массив или значение, — едва ли не самый практически важный навык JS для производительности Angular.

Мутирующие методы мутирует

Они меняют исходный массив на месте и опасны при OnPush / иммутабельном состоянии.

push(...items) мутирует

push — добавляет один или несколько элементов в конец массива.

  • Возвращает: новую length (число)
  • Параметры: добавляемые элементы
const a = [1, 2]; const len = a.push(3); // a = [1,2,3], len = 3

Подводный камень: this.orders.push(o) сохраняет ту же ссылку — OnPush не перерисует.

pop() мутирует

pop — удаляет и возвращает последний элемент.

  • Возвращает: удалённый элемент (или undefined, если массив пуст)
const a = [1, 2, 3]; const last = a.pop(); // a = [1,2], last = 3

shift() мутирует

shift — удаляет и возвращает первый элемент; переиндексирует остальные.

  • Возвращает: удалённый элемент
const a = [1, 2, 3]; const first = a.shift(); // a = [2,3], first = 1

Подводный камень: O(n) — все оставшиеся элементы переиндексируются. Избегайте в горячих циклах.

unshift(...items) мутирует

unshift — добавляет элементы в начало массива.

  • Возвращает: новую длину
const a = [2, 3]; a.unshift(0, 1); // a = [0,1,2,3]

splice(start, deleteCount, ...items) мутирует

splice — удаляет/заменяет/вставляет элементы в любой позиции. Швейцарский нож мутаций.

  • Возвращает: массив удалённых элементов
  • Параметры: индекс начала, количество для удаления, вставляемые элементы
const a = [1, 2, 3, 4];
const removed = a.splice(1, 2, 'x'); // a = [1,'x',4], removed = [2,3]

Не путайте с slice (не мутирует, копирует участок).

sort(compareFn?) мутирует

sort — сортирует массив на месте. Без компаратора элементы приводятся к строкам и сравниваются по UTF-16.

  • Возвращает: тот же (теперь отсортированный) массив
[10, 2, 1].sort();              // [1, 10, 2] ← string sort! gotcha
[10, 2, 1].sort((a, b) => a - b); // [1, 2, 10] ← numeric
// Immutable alternative (ES2023): array.toSorted((a,b) => a-b)

Подводный камень: сортировка по умолчанию лексикографическая — для чисел всегда передавайте компаратор.

reverse() мутирует

reverse — переворачивает массив на месте. (Иммутабельный вариант: toReversed().)

const a = [1, 2, 3]; a.reverse(); // a = [3,2,1]

fill(value, start?, end?) мутирует

fill — заполняет диапазон статическим значением.

new Array(3).fill(0);     // [0,0,0]
[1,2,3,4].fill(9, 1, 3);  // [1,9,9,4]

copyWithin(target, start, end?) мутирует

copyWithin — копирует участок массива в другую позицию того же массива, перезаписывая.

[1,2,3,4,5].copyWithin(0, 3); // [4,5,3,4,5]

Не мутирующие — возвращают новый массив новый массив

map(fn) новый массив

map — создаёт новый массив, преобразуя каждый элемент функцией.

  • Возвращает: новый массив той же длины
  • Параметры: (element, index, array)
const prices = [100, 200];
const withTax = prices.map(p => p * 1.2); // [120, 240]

filter(fn) новый массив

filter — создаёт новый массив только из элементов, прошедших предикат.

orders.filter(o => o.status === 'pending');

slice(start?, end?) новый массив

slice — возвращает поверхностную копию участка (конец не включается). Частая идиома иммутабельного копирования: arr.slice().

[1,2,3,4].slice(1, 3);  // [2,3]
[1,2,3,4].slice(-2);    // [3,4] — last 2

concat(...arrs) новый массив

concat — объединяет массивы/значения в новый массив.

[1,2].concat([3,4], 5); // [1,2,3,4,5]

flat(depth?) новый массив

flat — разворачивает вложенные массивы до глубины depth (по умолчанию 1). Используйте Infinity для полного разворота.

[1,[2,[3,[4]]]].flat(Infinity); // [1,2,3,4]

flatMap(fn) новый массив

flatMap — выполняет map, затем разворачивает на один уровень. Эффективнее, чем map().flat().

[1,2,3].flatMap(n => [n, n * 2]); // [1,2,2,4,3,6]

Array.from(iterable, mapFn?) новый массив

Array.from — создаёт новый массив из итерируемого или массивоподобного объекта (NodeList, Set, строка) с необязательной функцией отображения.

Array.from('abc');                       // ['a','b','c']
Array.from(new Set([1,1,2]));            // [1,2] — dedupe
Array.from({ length: 3 }, (_, i) => i);  // [0,1,2]

Не мутирующие — возвращают значение чистый

reduce(fn, seed?) & reduceRight чистый

reduce — сворачивает массив в одно накопленное значение слева направо. reduceRight делает то же справа налево.

  • Параметры: (accumulator, element, index, array) и начальное значение
const pnl = trades.reduce((sum, t) => sum + t.pnl, 0);
// group orders by symbol → Record
const bySymbol = orders.reduce((acc, o) => {
  (acc[o.symbol] ??= []).push(o);
  return acc;
}, {});

Подводный камень: всегда передавайте начальное значение — reduce на пустом массиве без него выбрасывает ошибку.

Поиск: find, findIndex, findLast, findLastIndex чистый

find — возвращает первый элемент, удовлетворяющий предикату (или undefined). findIndex возвращает его индекс (или -1). findLast / findLastIndex ищут с конца.

orders.find(o => o.id === 42);
orders.findIndex(o => o.status === 'filled');
orders.findLast(o => o.side === 'BUY');

Предикаты: some, every чистый

some — true, если проходит хотя бы один элемент. every — true, если проходят все (для пустого массива истинно по определению).

orders.some(o => o.status === 'error'); // any failures?
orders.every(o => o.qty > 0);           // all valid?

Поиск: includes, indexOf, lastIndexOf чистый

includes — true/false для наличия элемента (учитывает NaN). indexOf / lastIndexOf — первый/последний индекс или -1 (использует ===, пропускает NaN).

['BUY','SELL'].includes('BUY'); // true
[NaN].includes(NaN);            // true
[NaN].indexOf(NaN);             // -1 ← gotcha

join(sep?), at(index) чистый

join — объединяет элементы в строку с разделителем. at — возвращает элемент по индексу, поддерживая отрицательные значения (arr.at(-1) = последний).

['a','b','c'].join('-'); // 'a-b-c'
[10,20,30].at(-1);       // 30

forEach(fn) чистый

forEach — выполняет функцию для каждого элемента ради побочных эффектов. Возвращает undefined — нельзя выстроить в цепочку, нельзя сделать break.

orders.forEach(o => console.log(o.id)); // returns undefined

Нужно прервать раньше? Используйте цикл for...of или some().

Итераторы: entries, keys, values чистый

entries / keys / values — возвращают объекты-итераторы из пар [index, value], индексов или значений соответственно. Отлично сочетаются с for...of.

for (const [i, val] of ['a','b'].entries()) {
  console.log(i, val); // 0 'a' / 1 'b'
}

Почему изменяемость важна в Angular

Неправильно — та же ссылка на массив, OnPush пропускает изменение:

this.orders.push(newOrder); // mutation — reference unchanged, UI not updated

Правильно — новая ссылка, OnPush её обнаруживает:

this.orders = [...this.orders, newOrder]; // new array → CD runs → UI updates

То же самое правило управляет редьюсерами NgRx: никогда не мутируйте состояние, всегда возвращайте новый объект:

on(addOrder, (state, { order }) => ({
  ...state,
  orders: [...state.orders, order], // new array, new state object
}));

Методы объектов

Object.keys(obj) — массив собственных перечисляемых строковых ключей.

Object.values(obj) — массив собственных перечисляемых значений.

Object.entries(obj) — массив пар [key, value]. Сочетается с деструктуризацией в for...of.

Object.assign(target, ...sources) — копирует собственные перечисляемые свойства в target (поверхностно). Возвращает target.

Object.freeze(obj) — делает объект неизменяемым (поверхностно). Дальнейшие записи молча игнорируются (выбрасывают ошибку в строгом режиме).

Object.isFrozen(obj) — булево значение, заморожен ли объект.

Object.create(proto) — создаёт новый объект с заданным прототипом.

Object.defineProperty(obj, key, descriptor) — определяет свойство с тонким контролем (writable, enumerable, configurable, геттеры/сеттеры).

Object.getPrototypeOf(obj) — возвращает прототип. Предпочтительнее __proto__.

structuredClone(obj) — глубокое клонирование с поддержкой вложенных объектов, массивов, Map, Set, Date (но не функций/DOM).

obj.hasOwnProperty(key) — true только для собственных (не унаследованных) свойств. Безопаснее: Object.hasOwn(obj, key).

key in obj — true, если свойство существует в объекте или где-либо выше по цепочке прототипов.

Опциональная цепочка ?. — короткое замыкание в undefined вместо выбрасывания ошибки, когда звено null/undefined.

Нулевое слияние ?? — возвращает правую часть, только когда левая равна null или undefined (в отличие от ||, сохраняет 0, '', false).

const quote = { symbol: 'EURUSD', bid: 1.07, ask: 1.0702 };
Object.keys(quote);    // ['symbol','bid','ask']
Object.entries(quote); // [['symbol','EURUSD'], ['bid',1.07], ['ask',1.0702]]

const spread = quote?.ask ?? 0;   // 1.0702 (?. + ??)
const depth = quote?.book?.level1; // undefined, no throw

const frozen = Object.freeze({ rate: 1.07 });
frozen.rate = 2; // ignored (throws in strict mode)
Object.isFrozen(frozen); // true

?? против ||: price ?? 100 сохраняет настоящий 0; price || 100 ошибочно заменил бы 0 на 100.

Прототипы и цепочка прототипов

Прототип — у каждого объекта есть скрытая ссылка [[Prototype]] на другой объект. Когда вы читаете свойство, JS поднимается вверх по этой цепочке, пока не найдёт свойство или не достигнет null.

Цепочка наглядно

const arr = [1, 2, 3];
// arr → Array.prototype → Object.prototype → null
//        (map, filter…)    (hasOwnProperty…)
Object.getPrototypeOf(arr) === Array.prototype;            // true
Object.getPrototypeOf(Array.prototype) === Object.prototype; // true
Object.getPrototypeOf(Object.prototype);                  // null

Доступ к прототипу и его установка

__proto__ работает, но это устаревший аксессор — предпочитайте Object.getPrototypeOf() / Object.setPrototypeOf(). Создавайте объект с выбранным прототипом через Object.create(proto).

hasOwnProperty против in

const o = { a: 1 };
'a' in o;                 // true  (own)
'toString' in o;          // true  (inherited from Object.prototype)
o.hasOwnProperty('a');    // true
o.hasOwnProperty('toString'); // false ← only own properties

instanceof

instanceof — проверяет, появляется ли .prototype конструктора где-либо в цепочке прототипов объекта.

[] instanceof Array;  // true
[] instanceof Object; // true (Array.prototype's chain includes Object.prototype)

ES6 class — синтаксический сахар над прототипами

class — это синтаксический сахар над механизмом прототипов. extends выстраивает цепочку прототипов, super() вызывает родительский конструктор (обязателен в подклассе перед использованием this), static-члены живут на самом классе, а #field объявляет по-настоящему приватное поле.

// --- Prototype function pattern ---
function Instrument(symbol) { this.symbol = symbol; }
Instrument.prototype.describe = function () { return 'Instr: ' + this.symbol; };

// --- Equivalent class syntax ---
class Instrument2 {
  #internalId = Math.random();        // truly private
  static exchange = 'NASDAQ';         // on the class, not instances
  constructor(symbol) { this.symbol = symbol; }
  describe() { return 'Instr: ' + this.symbol; }
}
class Stock extends Instrument2 {
  constructor(symbol, sector) {
    super(symbol);                    // required before using `this`
    this.sector = sector;
  }
}

Spread, rest и деструктуризация

Spread ... — разворачивает итерируемый объект или объект в отдельные элементы или свойства.

Rest ... — собирает оставшиеся элементы/аргументы в один массив/объект. Тот же синтаксис, обратное направление.

Деструктуризация — распаковывает значения из массивов или свойства из объектов в отдельные переменные.

// Spread — arrays & objects
const merged = [...[1, 2], ...[3, 4]];          // [1,2,3,4]
const updated = { ...order, status: 'filled' };  // override one field

// Rest params
function sum(...nums) { return nums.reduce((a, b) => a + b, 0); }

// Array destructuring (with rest)
const [first, second, ...others] = [10, 20, 30, 40]; // others = [30,40]

// Object destructuring (rename + default)
const { name, age: userAge = 25 } = { name: 'EURUSD' }; // userAge = 25

// Nested destructuring
const { meta: { source } } = { meta: { source: 'WS' } }; // source = 'WS'

// In function params (a senior-favourite pattern)
function place({ symbol, qty = 1, side = 'BUY' }) { /* ... */ }
place({ symbol: 'BTCUSD', qty: 5 });

Замыкания и всплытие

var против let против const и TDZ

Всплытие (hoisting) — объявления концептуально поднимаются в начало своей области видимости до выполнения. var всплывает и инициализируется значением undefined; let/const всплывают, но не инициализируются, создавая временную мёртвую зону (TDZ) — обращение к ним до объявления выбрасывает ошибку.

console.log(a); // undefined — var hoisted & initialized
var a = 1;

console.log(b); // ❌ ReferenceError — TDZ
let b = 2;

Замыкания

Замыкание — функция, которая запоминает и может обращаться к переменным из своей внешней области видимости даже после того, как внешняя функция завершилась.

// 1. Counter factory
function makeCounter() {
  let count = 0;                 // private, captured
  return () => ++count;
}
const next = makeCounter();
next(); next(); // 1, 2

// 2. Private state (module pattern)
const priceStore = (() => {
  let price = 0;                 // not accessible from outside
  return {
    set: (p) => { price = p; },
    get: () => price,
  };
})();

// 3. Memoization
function memoize(fn) {
  const cache = new Map();        // captured across calls
  return (n) => cache.has(n) ? cache.get(n) : (cache.set(n, fn(n)), cache.get(n));
}

Классическая ловушка цикла

// ❌ var — all callbacks share ONE i, prints 3, 3, 3
for (var i = 0; i < 3; i++) setTimeout(() => console.log(i), 0);

// ✅ let — block-scoped i, fresh binding each iteration → 0, 1, 2
for (let i = 0; i < 3; i++) setTimeout(() => console.log(i), 0);

// ✅ IIFE fix for var
for (var j = 0; j < 3; j++) ((k) => setTimeout(() => console.log(k), 0))(j);

Контекст Angular: сервисы используют замыкания для инкапсуляции приватного состояния — например, BehaviorSubject, захваченный в поле класса и доступный только через публичные методы.

Цикл событий

Стек вызовов — LIFO-стек, выполняющий синхронный код по одному фрейму за раз.

Web API — предоставляемые браузером средства (setTimeout, fetch, события DOM), которые работают вне стека и ставят колбэки в очередь по завершении.

Очередь микрозадач — содержит колбэки Promise.then, queueMicrotask, MutationObserver. Полностью опустошается перед следующей макрозадачей и после каждой из них.

Очередь макрозадач — содержит колбэки setTimeout/setInterval, ввод-вывод, UI-события. За один тик цикла выполняется одна макрозадача.

Разбор — почему порядок 1, 4, 3, 2

console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1, 4, 3, 2
  1. console.log('1') выполняется синхронно → 1.
  2. setTimeout передаёт свой колбэк в Web API; колбэк ставится в очередь как макрозадача.
  3. Promise.resolve().then ставит свой колбэк в очередь как микрозадачу.
  4. console.log('4') выполняется синхронно → 4.
  5. Стек теперь пуст. Цикл событий опустошает сначала все микрозадачи → 3.
  6. Только потом берётся одна макрозадача → 2.

Promise и async/await

Promise проходит через состояния: pending → fulfilled (разрешён значением) или rejected (с причиной). После урегулирования он больше никогда не меняется.

.then(onFulfilled) — выполняется, когда промис разрешается, получает значение и возвращает новый промис (что и позволяет строить цепочки).

.catch(onRejected) — выполняется, когда промис (или любой более ранний шаг) отклоняется.

.finally(fn) — выполняется всегда при урегулировании, не получает аргументов. Идеально для очистки, например скрытия спиннера.

fetchOrder(42)
  .then(order => order.symbol)      // returns a value → next .then gets it
  .then(symbol => fetchQuote(symbol)) // returns a promise → awaited
  .then(quote => console.log(quote.bid))
  .catch(err => console.error('failed:', err))
  .finally(() => hideSpinner());

async / await

async-функция эквивалентна цепочке .then, но читается как синхронный код. await приостанавливает только текущую async-функцию — остальная программа продолжает работать. Оборачивайте в try/catch для обработки ошибок (в TS пойманная ошибка имеет тип unknown, поэтому её нужно сузить).

async function loadDashboard(): Promise<void> {
  try {
    const order = await fetchOrder(42);
    const quote = await fetchQuote(order.symbol);
    render(quote);
  } catch (err: unknown) {                 // unknown, not any
    if (err instanceof Error) console.error(err.message);
  } finally {
    hideSpinner();
  }
}

this, call, apply, bind

this определяется тем, как функция вызвана, а не где она определена (кроме стрелочных функций).

call(thisArg, ...args) — немедленно вызывает функцию с заданным this и аргументами, переданными по отдельности.

apply(thisArg, argsArray) — то же, что call, но аргументы передаются массивом.

bind(thisArg, ...args) — возвращает новую функцию с навсегда привязанным this; саму функцию не вызывает.

function greet(greeting) { return greeting + ', ' + this.symbol; }
const ctx = { symbol: 'EURUSD' };

greet.call(ctx, 'Hi');        // 'Hi, EURUSD'
greet.apply(ctx, ['Hello']);  // 'Hello, EURUSD'
const bound = greet.bind(ctx);
bound('Hey');                 // 'Hey, EURUSD'

Стрелочные функции не имеют собственного this — они наследуют его лексически. Именно поэтому их используют для методов класса в Angular и колбэков RxJS: this остаётся экземпляром компонента, а не становится undefined.

Вопросы и ответы — вопросы с собеседований

1. В чём разница между == и ===?

=== (строгое равенство) сравнивает значение и тип без приведения. == (нестрогое равенство) сначала приводит операнды к общему типу, что даёт сюрпризы вроде 0 == '' → true, null == undefined → true, [] == false → true. Всегда используйте ===, кроме идиоматического x == null для отлова и null, и undefined.

2. Что такое временная мёртвая зона (TDZ)?

Промежуток между входом в область видимости и точкой, где объявлена переменная let/const. Переменная всплыла, но не инициализирована, поэтому любое обращение в этом промежутке выбрасывает ReferenceError. TDZ существует, чтобы ловить ошибки использования до объявления, которые var молча допускал (возвращая undefined).

3. Почему typeof null === 'object'?

Ошибка из оригинального движка JS 1995 года. Значения помечались битовым шаблоном типа; объекты использовали тег 0, а null был представлен нулевым указателем 0x00, который тоже читался как тег 0. Исправление сломало бы слишком много существующего кода, поэтому это остаётся причудой языка.

4. Каков вывод setTimeout(fn, 0) против Promise.resolve().then(fn)?

Колбэк Promise выполняется первым. Promise.then планирует микрозадачу; setTimeout планирует макрозадачу. После завершения синхронного кода цикл событий опустошает всю очередь микрозадач, прежде чем взять любую макрозадачу — поэтому колбэк промиса всегда побеждает, даже с таймаутом в 0 мс.

5. Что такое замыкание? Приведите практический пример.

Функция в связке со ссылками на окружающую лексическую область — она сохраняет доступ к внешним переменным даже после того, как внешняя функция завершилась. Практическое применение: приватное состояние (фабрика счётчиков или модульный паттерн), кэши мемоизации и фабрики обработчиков событий. В Angular сервис часто замыкает приватный BehaviorSubject и предоставляет только методы чтения.

Тест

1. Какой метод массива мутирует исходный массив?
sort() (а также reverse, splice, push, pop, shift, unshift, fill, copyWithin) мутируют на месте. map, filter, slice возвращают новые массивы.
2. Что возвращает [10, 2, 1].sort()?
Сортировка по умолчанию приводит элементы к строкам — "10" < "2" лексикографически. Передайте компаратор: sort((a,b) => a-b).
3. Что будет выведено? console.log(1); setTimeout(()=>console.log(2)); Promise.resolve().then(()=>console.log(3)); console.log(4);
Сначала синхронно (1, 4), затем микрозадачи (Promise → 3), затем макрозадачи (setTimeout → 2).
4. Что корректно обновляет массив так, чтобы OnPush обнаружил изменение?
OnPush проверяет ссылки. Только присваивание совершенно нового массива создаёт новую ссылку, которую видит обнаружение изменений.
