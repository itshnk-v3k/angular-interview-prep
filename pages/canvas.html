Canvas API и графики

Отрисовка высокочастотных торговых графиков в браузере: Canvas 2D API, полноценный рендерер свечей и объёмов, а также приёмы производительности (rAF, частичная перерисовка, devicePixelRatio, OffscreenCanvas), которые держат живой ценовой график плавным на 60 fps.

Canvas против SVG против WebGL

Три способа отрисовки графики в вебе. Выбор правильного — первый вопрос, который задаёт интервьюер про построение графиков: какая технология и почему.

Canvas — растровая поверхность непосредственного режима (immediate-mode). Вы отдаёте команды рисования, и они закрашивают пиксели на битмапе. Здесь нет сохраняемых объектов и нет DOM-узлов — после отрисовки фигура остаётся лишь набором пикселей. Идеально для тысяч часто обновляющихся элементов, например ценовых графиков.

SVG — векторный формат сохраняемого режима (retained-mode). Каждая фигура — это настоящий DOM-элемент, который можно стилизовать через CSS, навешивать на него события и инспектировать. Не зависит от разрешения, но производительность падает, когда в документе тысячи узлов.

WebGL — низкоуровневый API с GPU-ускорением (на базе OpenGL ES). Рендерит через шейдеры, выполняемые на видеокарте; лучший выбор для огромных наборов данных, 3D и тяжёлых визуальных эффектов ценой гораздо большей сложности.

Эмпирическое правило: несколько интерактивных фигур, которым нужны доступность и CSS → SVG. Живая тепловая карта стакана заявок или график свечей, перерисовывающийся на каждом тике → Canvas. Десятки миллионов точек или 3D-поверхности глубины → WebGL.

Основы Canvas API

Получение контекста

Контекст рендеринга — объект, открывающий доступ ко всем методам рисования. 2D-контекст получают через canvas.getContext('2d'). Он возвращает CanvasRenderingContext2D | null, поэтому всегда сужайте тип перед использованием.

const canvas = document.querySelector<HTMLCanvasElement>('#chart');
if (!canvas) throw new Error('canvas not found');

const ctx: CanvasRenderingContext2D | null = canvas.getContext('2d');
if (!ctx) throw new Error('2d context unavailable');

// ctx is now narrowed to CanvasRenderingContext2D
ctx.fillStyle = '#0a0';

Система координат

Система координат — начало координат (0, 0) находится в левом верхнем углу. x растёт вправо, y растёт вниз. Для графиков это важно: более высокие цены — это меньшие значения y, поэтому отображение цены в y должно инвертировать ось.

// (0,0) ──────────────▶ +x
//   │
//   │   canvas pixels
//   ▼
//  +y         a higher price → a SMALLER y (closer to the top)

Примитивы рисования

fillRect(x, y, w, h) — рисует залитый прямоугольник текущим fillStyle. Рабочая лошадка для тел свечей и столбцов объёма.

strokeRect(x, y, w, h) — рисует только контур прямоугольника, используя strokeStyle и lineWidth.

clearRect(x, y, w, h) — стирает прямоугольную область до прозрачности. Ключевой метод для перерисовки и частичного рендеринга.

beginPath() — начинает новый путь, отбрасывая все ранее записанные подпути. Вызывайте перед каждой независимой фигурой.

moveTo(x, y) — перемещает «перо» в точку без рисования, начиная новый подпуть.

lineTo(x, y) — добавляет прямой отрезок от текущей точки до (x, y).

arc(x, y, r, start, end, ccw?) — добавляет дугу окружности; полная окружность задаётся от 0 до 2 * Math.PI. Используется для маркеров и точек.

quadraticCurveTo(cpx, cpy, x, y) — добавляет квадратичную кривую Безье с одной контрольной точкой.

bezierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y) — добавляет кубическую кривую Безье с двумя контрольными точками; более гладкие линейные графики.

fill() — заливает текущий путь цветом fillStyle.

stroke() — обводит текущий путь цветом strokeStyle.

// Filled candle body
ctx.fillStyle = '#16a34a';
ctx.fillRect(40, 100, 8, 60);

// A wick (high → low) as a stroked line
ctx.strokeStyle = '#16a34a';
ctx.lineWidth = 1;
ctx.beginPath();
ctx.moveTo(44, 80);
ctx.lineTo(44, 180);
ctx.stroke();

// A smooth line-chart segment with a Bézier curve
ctx.beginPath();
ctx.moveTo(0, 200);
ctx.bezierCurveTo(50, 120, 100, 260, 150, 180);
ctx.stroke();

// A circular marker
ctx.beginPath();
ctx.arc(150, 180, 3, 0, 2 * Math.PI);
ctx.fill();

Рисование текста

fillText(text, x, y) — рисует залитый текст, используя font, fillStyle, textAlign и textBaseline. Применяется для подписей осей и ценовых меток.

strokeText(text, x, y) — рисует только контур текста.

ctx.font = '12px monospace';
ctx.fillStyle = '#475569';
ctx.textAlign = 'right';
ctx.textBaseline = 'middle';
ctx.fillText('1.0850', 60, 100); // a price-axis label

Рисование изображений

drawImage(source, dx, dy, dw?, dh?) — рисует изображение, кадр видео или другой canvas на текущем, при необходимости масштабируя. Именно так OffscreenCanvas или закешированный слой переносится на видимый canvas.

// Blit a pre-rendered grid layer onto the main canvas
ctx.drawImage(gridLayer, 0, 0);

// Draw a scaled logo watermark
const logo = new Image();
logo.onload = () => ctx.drawImage(logo, 10, 10, 48, 48);
logo.src = '/assets/mark.png';

Трансформации

translate(dx, dy) — сдвигает начало системы координат. Рисование в (0,0) после этого попадёт в (dx, dy).

rotate(angle) — поворачивает систему координат на angle радиан вокруг текущего начала. Используется для повёрнутых подписей оси времени.

scale(sx, sy) — масштабирует последующее рисование. Передача devicePixelRatio по обеим осям даёт чёткий вывод на Retina.

save() — кладёт текущее состояние рисования (трансформация, стили, клип) в стек.

restore() — снимает последнее сохранённое состояние, отменяя все трансформации и изменения стилей с момента соответствующего save(). Всегда используйте их в паре.

ctx.save();                 // remember the clean state
ctx.translate(80, 200);     // move origin to the label anchor
ctx.rotate(-Math.PI / 4);   // tilt 45° for a dense time axis
ctx.fillText('09:30', 0, 0);
ctx.restore();              // undo translate + rotate

Подводный камень: трансформации накапливаются. Если забыть restore() после translate/rotate, весь canvas окажется смещён на следующем кадре. Всегда оборачивайте трансформированное рисование в пару save()/restore().

График свечей

Свеча — один бар OHLCV: цены открытия, максимума, минимума, закрытия и торговый объём за фиксированный интервал времени. Зелёная (бычья), когда close >= open, красная (медвежья) в остальных случаях.

Полноценный, хорошо прокомментированный рендерер, который рисует фитили свечей, тела и гистограмму объёмов из массива Candle[], отображая цену → y (с инверсией) и индекс → x.

interface Candle {
  time: number;   // epoch ms at the start of the interval
  open: number;
  high: number;
  low: number;
  close: number;
  volume: number;
}

interface ChartTheme {
  bull: string;   // colour for close >= open
  bear: string;   // colour for close < open
  volume: string;
  axis: string;
}

const DEFAULT_THEME: ChartTheme = {
  bull: '#16a34a',
  bear: '#dc2626',
  volume: 'rgba(100, 116, 139, 0.4)',
  axis: '#94a3b8',
};

class CandlestickChart {
  private readonly ctx: CanvasRenderingContext2D;

  // Layout: the price panel sits on top, the volume panel below it.
  private readonly padding = { top: 12, right: 64, bottom: 24, left: 12 };
  private readonly volumeRatio = 0.2; // volume panel takes 20% of the height

  constructor(
    private readonly canvas: HTMLCanvasElement,
    private readonly theme: ChartTheme = DEFAULT_THEME,
  ) {
    const ctx = canvas.getContext('2d');
    if (!ctx) throw new Error('2d context unavailable');
    this.ctx = ctx;
  }

  /** Draw the whole chart for the given candles. */
  render(candles: Candle[]): void {
    if (candles.length === 0) return;
    const { width, height } = this.canvas;
    const { top, right, bottom, left } = this.padding;

    // ---- Panel geometry ----
    const plotWidth = width - left - right;
    const plotHeight = height - top - bottom;
    const volumeHeight = plotHeight * this.volumeRatio;
    const priceHeight = plotHeight - volumeHeight;
    const priceTop = top;
    const volumeTop = top + priceHeight;

    // ---- Price scale: map a price to a y pixel (inverted axis) ----
    const highest = Math.max(...candles.map(c => c.high));
    const lowest = Math.min(...candles.map(c => c.low));
    const priceRange = highest - lowest || 1; // guard against flat series
    const priceToY = (price: number): number =>
      priceTop + (highest - price) / priceRange * priceHeight;

    // ---- Time axis: map a candle index to an x pixel ----
    const slot = plotWidth / candles.length;      // px per candle
    const bodyWidth = Math.max(1, slot * 0.6);    // leave a gap between candles
    const indexToX = (i: number): number => left + i * slot + slot / 2;

    // ---- Volume scale ----
    const maxVolume = Math.max(...candles.map(c => c.volume)) || 1;

    // Clear the frame before redrawing.
    this.ctx.clearRect(0, 0, width, height);

    // ---- Draw each candle ----
    for (let i = 0; i < candles.length; i++) {
      const candle = candles[i];
      const x = indexToX(i);
      const bullish = candle.close >= candle.open;
      const colour = bullish ? this.theme.bull : this.theme.bear;

      // Wick: a vertical line from high to low.
      this.ctx.strokeStyle = colour;
      this.ctx.lineWidth = 1;
      this.ctx.beginPath();
      this.ctx.moveTo(x, priceToY(candle.high));
      this.ctx.lineTo(x, priceToY(candle.low));
      this.ctx.stroke();

      // Body: a filled rectangle between open and close.
      const yOpen = priceToY(candle.open);
      const yClose = priceToY(candle.close);
      const bodyTop = Math.min(yOpen, yClose);
      const bodyHeight = Math.max(1, Math.abs(yClose - yOpen)); // min 1px (doji)
      this.ctx.fillStyle = colour;
      this.ctx.fillRect(x - bodyWidth / 2, bodyTop, bodyWidth, bodyHeight);

      // Volume bar at the bottom panel.
      const vH = candle.volume / maxVolume * volumeHeight;
      this.ctx.fillStyle = this.theme.volume;
      this.ctx.fillRect(x - bodyWidth / 2, volumeTop + volumeHeight - vH, bodyWidth, vH);
    }

    // ---- Price axis labels on the right ----
    this.drawPriceAxis(highest, lowest, priceToY, width - right + 6);
  }

  /** Draw a handful of evenly-spaced price labels. */
  private drawPriceAxis(
    highest: number,
    lowest: number,
    priceToY: (p: number) => number,
    labelX: number,
  ): void {
    const ticks = 5;
    this.ctx.fillStyle = this.theme.axis;
    this.ctx.font = '11px monospace';
    this.ctx.textAlign = 'left';
    this.ctx.textBaseline = 'middle';
    for (let t = 0; t <= ticks; t++) {
      const price = lowest + (highest - lowest) * (t / ticks);
      this.ctx.fillText(price.toFixed(4), labelX, priceToY(price));
    }
  }
}

// Usage
const el = document.querySelector<HTMLCanvasElement>('#chart');
if (el) {
  const chart = new CandlestickChart(el);
  chart.render(candles); // candles: Candle[]
}

Почему priceToY инвертирует: ось y у canvas растёт вниз, а цены растут вверх. Формула (highest - price) / range отображает самую высокую цену в верх (малое y), а самую низкую — в низ (большое y).

Обработанные краевые случаи: плоский ряд (priceRange откатывается к 1), доджи, где открытие равно закрытию (тело принудительно делается 1px, чтобы оставаться видимым), и нулевой объём (maxVolume откатывается к 1).

Производительность

requestAnimationFrame

requestAnimationFrame(cb) — планирует запуск cb прямо перед следующей перерисовкой браузера (~60 fps, в такт частоте обновления дисплея). Колбэк получает высокоточную метку времени. Предпочитайте его вместо setTimeout для анимации: он приостанавливается в фоновых вкладках, синхронизируется с дисплеем и избегает пропущенных или сдвоенных кадров.

class RenderLoop {
  private rafId = 0;
  private running = false;

  constructor(private readonly draw: (timestamp: number) => void) {}

  start(): void {
    if (this.running) return;
    this.running = true;
    const frame = (timestamp: number): void => {
      if (!this.running) return;
      this.draw(timestamp);                       // render one frame
      this.rafId = requestAnimationFrame(frame);  // queue the next
    };
    this.rafId = requestAnimationFrame(frame);
  }

  stop(): void {
    this.running = false;
    cancelAnimationFrame(this.rafId);
  }
}

// Drive the chart from live quotes
const loop = new RenderLoop(() => chart.render(latestCandles));
loop.start();
// loop.stop(); // call on ngOnDestroy

Замечание про Angular: запускайте цикл через NgZone.runOutsideAngular(), чтобы каждый кадр не запускал обнаружение изменений, и всегда вызывайте cancelAnimationFrame в ngOnDestroy, чтобы избежать утечки цикла.

Частичная перерисовка

Частичная перерисовка (dirty rendering) — вместо очистки и перерисовки всего canvas на каждом кадре перерисовывайте только изменившуюся область («грязный прямоугольник»). На широком графике, где на каждом тике обновляется лишь новейшая свеча, это резко сокращает работу на кадр.

// Only the last candle moves on each tick → repaint just its column.
function redrawLastCandle(
  ctx: CanvasRenderingContext2D,
  column: { x: number; width: number; top: number; height: number },
  drawColumn: () => void,
): void {
  // Clear ONLY the dirty rectangle, not the entire canvas.
  ctx.clearRect(column.x, column.top, column.width, column.height);
  drawColumn(); // repaint that single column
}

Альтернатива со слоями: держите статичный контент (сетку, оси) на фоновом canvas, нарисованном один раз, а на прозрачном верхнем canvas — только динамический ряд. Дорогой слой никогда не перерисовывается.

devicePixelRatio

devicePixelRatio (DPR) — отношение физических пикселей устройства к CSS-пикселям. На Retina/HiDPI-дисплее оно равно 2 (или больше). Canvas, размер которого задан только в CSS, рендерится размыто; нужно задать его буфер отрисовки в размере cssSize × DPR и масштабировать контекст, чтобы одна единица рисования соответствовала одному физическому пикселю.

function setupHiDpiCanvas(
  canvas: HTMLCanvasElement,
  cssWidth: number,
  cssHeight: number,
): CanvasRenderingContext2D {
  const dpr = window.devicePixelRatio || 1;

  // Backing store in physical pixels (sharp).
  canvas.width = Math.round(cssWidth * dpr);
  canvas.height = Math.round(cssHeight * dpr);

  // Display size stays in CSS pixels.
  canvas.style.width = `${cssWidth}px`;
  canvas.style.height = `${cssHeight}px`;

  const ctx = canvas.getContext('2d');
  if (!ctx) throw new Error('2d context unavailable');

  // Scale so all drawing code can use CSS-pixel coordinates.
  ctx.scale(dpr, dpr);
  return ctx;
}

Подводный камень: если задать canvas.width/height, но забыть canvas.style.width/height (или наоборот), график будет либо размытым, либо неправильно растянутым. Обе пары должны быть заданы, а scale(dpr, dpr) применён один раз после установки размеров.

OffscreenCanvas

OffscreenCanvas — canvas, отвязанный от DOM, который можно рендерить из Web Worker. Тяжёлая отрисовка графика происходит вне основного потока, сохраняя UI отзывчивым; результат передаётся на видимый canvas через transferControlToOffscreen().

// --- main thread ---
const canvas = document.querySelector<HTMLCanvasElement>('#chart');
if (!canvas) throw new Error('canvas not found');

const offscreen: OffscreenCanvas = canvas.transferControlToOffscreen();
const worker = new Worker(new URL('./chart.worker', import.meta.url), { type: 'module' });

// Transfer ownership of the offscreen canvas to the worker (zero-copy).
worker.postMessage({ type: 'init', canvas: offscreen }, [offscreen]);

// Stream live candles to the worker for rendering.
worker.postMessage({ type: 'candles', candles: latestCandles });

// --- chart.worker.ts ---
let ctx: OffscreenCanvasRenderingContext2D | null = null;

onmessage = (e: MessageEvent): void => {
  const data = e.data as
    | { type: 'init'; canvas: OffscreenCanvas }
    | { type: 'candles'; candles: Candle[] };

  if (data.type === 'init') {
    ctx = data.canvas.getContext('2d');
  } else if (data.type === 'candles' && ctx) {
    drawCandles(ctx, data.candles); // runs entirely off the main thread
  }
};

Когда это окупается: плотные графики (тысячи свечей) или несколько панелей, перерисовывающихся на каждом тике. Для одного лёгкого графика накладные расходы на обмен с воркером и сериализацию могут перевесить выгоду — сначала измеряйте.

Вопросы и ответы — собеседование

1. Когда вы выберете Canvas вместо SVG для графика?

Когда есть много часто обновляющихся элементов — живой график свечей, тепловая карта стакана заявок, тысячи точек. SVG создаёт DOM-узел на каждую фигуру, поэтому график с тысячами элементов раздувает документ, и узким местом становится перекомпоновка/перерисовка. Canvas — это растр непосредственного режима: он просто закрашивает пиксели, поэтому перерисовка тысяч свечей на каждом тике остаётся дешёвой. Компромисс в том, что у Canvas нет DOM — нет стилизации через CSS, нет обработчиков событий на фигурах, слабее доступность. SVG выигрывает для небольшого числа интерактивных, стилизуемых и доступных фигур (иконки, одна аннотированная линия).

2. Почему requestAnimationFrame предпочтительнее setTimeout для анимации?

requestAnimationFrame синхронизирован с частотой обновления дисплея (~60 fps) и срабатывает прямо перед следующей перерисовкой, поэтому кадры никогда не пропускаются и не дублируются, а анимация выглядит плавной. Он автоматически приостанавливается в фоновых или скрытых вкладках, экономя CPU и батарею. setTimeout(fn, 16) дрейфует относительно реальной частоты обновления, продолжает срабатывать в фоновых вкладках и может рисовать частичные кадры. rAF также передаёт высокоточную метку времени, которую можно использовать для вычисления дельт по времени.

3. Почему canvas выглядит размытым на экране Retina и как это исправить?

На HiDPI-дисплее devicePixelRatio равен 2 или больше: каждый CSS-пиксель соответствует нескольким физическим. Если буфер отрисовки canvas задан только в CSS-пикселях, браузер увеличивает битмап, и он выглядит мягким. Решение: задать canvas.width = cssWidth * dpr и canvas.height = cssHeight * dpr для буфера отрисовки, оставить canvas.style.width/height равными CSS-размеру, затем один раз вызвать ctx.scale(dpr, dpr), чтобы код рисования мог продолжать использовать CSS-пиксельные координаты. Теперь одна единица рисования равна одному физическому пикселю, и линии чёткие.

4. Что такое OffscreenCanvas и какую проблему он решает?

Это canvas, отвязанный от DOM, который можно рендерить из Web Worker. Вы вызываете transferControlToOffscreen() на видимом canvas и через postMessage передаёте offscreen-дескриптор воркеру (как transferable, поэтому без копирования). Воркер затем выполняет всю дорогую отрисовку вне основного потока, поэтому прокрутка, ввод и обнаружение изменений Angular остаются отзывчивыми даже во время перерисовки плотного графика. Это решает проблему конкуренции за основной поток; ценой является настройка воркера и сериализация сообщений, поэтому это окупается в основном для тяжёлых или многопанельных графиков — измеряйте перед внедрением.

Тест

1. Вам нужен ценовой график, перерисовывающий тысячи свечей на каждом тике. Какая технология подходит лучше всего?
Canvas закрашивает пиксели без создания DOM-узлов, поэтому перерисовка тысяч часто обновляющихся свечей остаётся дешёвой. SVG/div создают по узлу на фигуру и захлёбываются на перекомпоновке.
2. Где в системе координат canvas находится начало координат (0, 0)?
Начало координат canvas — вверху слева; x растёт вправо, y растёт вниз. Именно поэтому отображение цены→y должно инвертировать ось, помещая более высокие цены в меньшие значения y.
3. Чтобы отрисовать чёткий canvas на экране с devicePixelRatio === 2, нужно:
Задайте буфер отрисовки в размере cssSize × dpr, оставьте CSS-размер для отображения, затем один раз масштабируйте контекст на dpr, чтобы код рисования продолжал использовать CSS-пиксельные координаты. Тогда одна единица равна одному физическому пикселю.
