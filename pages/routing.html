Маршрутизация (Router)

Router в Angular сопоставляет URL-адреса с деревьями компонентов: конфигурация, навигация, параметры маршрута, ленивая загрузка, гарды, резолверы и события — плюс CDK BreakpointObserver для адаптивных макетов.

Конфигурация

Массив Routes сопоставляет пути URL с компонентами или лениво загружаемыми чанками. Вы регистрируете его либо через API NgModule (классический подход), либо через standalone-API provideRouter (современный подход).

RouterModule.forRoot(routes) — вызывается один раз в корневом модуле приложения. Создаёт singleton-сервис Router, регистрирует маршруты верхнего уровня и настраивает глобальные параметры (предзагрузку, восстановление прокрутки, навигацию в памяти).

RouterModule.forChild(routes) — используется в функциональных модулях. Регистрирует дополнительные маршруты, но не создаёт сервис Router повторно, избегая дублирования экземпляров провайдера.

provideRouter(routes) — современный standalone-API для начальной загрузки (Angular 14+). Передаётся в bootstrapApplication через ApplicationConfig; заменяет RouterModule.forRoot компонуемыми возможностями, такими как withPreloading, withComponentInputBinding, withRouterConfig.

Стиль NgModule (классический)

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  { path: 'orders', component: OrderListComponent },
  { path: 'instruments/:symbol', component: InstrumentComponent },
  { path: '', redirectTo: 'orders', pathMatch: 'full' },
  { path: '**', component: NotFoundComponent },
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],   // once, at the root
  exports: [RouterModule],
})
export class AppRoutingModule {}

// Feature module
@NgModule({
  imports: [RouterModule.forChild(featureRoutes)], // child routes
})
export class OrdersRoutingModule {}

Стиль standalone (современный)

import { bootstrapApplication } from '@angular/platform-browser';
import { provideRouter, withPreloading, PreloadAllModules } from '@angular/router';

bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes, withPreloading(PreloadAllModules)),
  ],
});

Свойства маршрута

Каждый объект Route настраивается через эти свойства:

path — сегмент URL для сопоставления ('orders', 'instruments/:symbol', '' пустой или '**' wildcard). Без ведущего слеша.

component — компонент, который отрисовывается при совпадении пути.

loadChildren — функция, возвращающая динамический import() лениво загружаемого модуля (или массив дочерних маршрутов).

loadComponent — функция, возвращающая динамический import() standalone-компонента (ленивая загрузка на уровне компонента).

children — массив вложенных дочерних маршрутов, отрисовываемых в дочерний <router-outlet>.

redirectTo — путь, на который происходит переадресация при совпадении этого маршрута.

pathMatch — 'full' (весь URL должен равняться пути) или 'prefix' (по умолчанию; путь является префиксом). Обязателен для переадресаций с пустым путём.

data — статический неизменяемый объект произвольных метаданных, доступный через ActivatedRoute.data (например, { roles: ['trader'] }).

resolve — карта резолверов, которые предзагружают данные до активации маршрута.

canActivate — гарды, решающие, разрешена ли навигация к этому маршруту.

canActivateChild — гарды, применяемые ко всем дочерним маршрутам этого маршрута.

canDeactivate — гарды, решающие, разрешена ли навигация прочь от этого маршрута.

canLoad — гарды, решающие, можно ли загрузить ленивый модуль (устаревший; заменён на canMatch).

canMatch — гарды, решающие, совпадает ли маршрут вообще; позволяет роутеру пробовать альтернативные маршруты, когда возвращается false.

outlet — имя именованного <router-outlet>, в который выполняется отрисовка (для вторичных/вспомогательных маршрутов).

title — строка или резолвер, устанавливающий document.title при активации маршрута.

Навигация

Навигацию выполняют либо декларативно из шаблона с помощью routerLink, либо императивно из класса с помощью сервиса Router.

routerLink — директива, связывающая элемент с маршрутом. Принимает строку или «массив параметров ссылки» из сегментов пути и параметров.

routerLinkActive — добавляет CSS-класс(ы) к элементу, пока маршрут его ссылки активен. Используйте [routerLinkActiveOptions]="{ exact: true }" для точного совпадения.

Декларативно — ссылки в шаблоне

<!-- Static path -->
<a routerLink="/orders" routerLinkActive="active-class">Orders</a>

<!-- Dynamic segment via parameters array -->
<a [routerLink]="['/orders', order.id]">View order</a>

<!-- With query params and a fragment -->
<a [routerLink]="['/instruments', symbol]"
   [queryParams]="{ tf: '1h' }"
   fragment="depth"
   routerLinkActive="active-class">
  {{ symbol }}
</a>

Router.navigate(commands, extras?) — императивно навигирует с использованием массива параметров; возвращает Promise<boolean>.

Router.navigateByUrl(url, extras?) — императивно навигирует с использованием строки абсолютного URL (игнорирует текущий маршрут).

Императивно — сервис Router

import { Component, inject } from '@angular/core';
import { Router, ActivatedRoute } from '@angular/router';

@Component({ /* ... */ })
export class OrderActionsComponent {
  private readonly router = inject(Router);
  private readonly route = inject(ActivatedRoute);

  openOrder(id: string): void {
    // Absolute, parameters array
    this.router.navigate(['/orders', id]);
  }

  goToSibling(): void {
    // Relative navigation — resolves against the current route
    this.router.navigate(['../sibling'], { relativeTo: this.route });
  }

  openByUrl(): void {
    // Absolute URL string
    this.router.navigateByUrl('/orders/123');
  }
}

Параметры навигации (NavigationExtras)

queryParams — объект параметров строки запроса для добавления (?tf=1h&side=BUY).

fragment — хеш-фрагмент URL (#depth), часто используется для якорей внутри страницы.

replaceUrl — при значении true заменяет текущую запись истории вместо добавления новой (без записи для кнопки «Назад»).

skipLocationChange — навигирует и отрисовывает цель, но не меняет URL в браузере.

queryParamsHandling — 'merge' (объединить новые с текущими), 'preserve' (сохранить текущие, игнорировать новые) или null (заменить, по умолчанию).

this.router.navigate(['/instruments', symbol], {
  queryParams: { tf: '1h' },
  queryParamsHandling: 'merge',
  fragment: 'depth',
  replaceUrl: true,
});

Совет по относительной навигации: всегда передавайте { relativeTo: this.route } с относительными командами вроде ['../sibling'] — без этого роутер разрешает путь относительно корня, и вы получаете неправильный URL.

ActivatedRoute

ActivatedRoute — внедряемый сервис, предоставляющий информацию о маршруте, связанном с текущим отрисованным компонентом: его параметры, параметры запроса, статические/резолвленные данные и сегменты URL — в основном в виде Observable, чтобы компонент реагировал на изменения.

params — Observable<Params> параметров пути маршрута. Излучает снова, когда меняются только параметры, но компонент переиспользуется (например, /orders/1 → /orders/2). Предпочитайте paramMap для типизированного API карты.

queryParams — Observable<Params> параметров строки запроса URL.

data — Observable<Data>, объединяющий статические route.data со всем, что возвращают резолверы.

snapshot.params — синхронный снимок параметров на момент активации. Удобно, но он не обновляется при переиспользовании компонента, поэтому используйте его только тогда, когда компонент не может быть переиспользован для того же маршрута.

ActivatedRouteSnapshot — неизменяемый снимок состояния маршрута на конкретный момент времени (params, queryParams, data, url) — также передаётся в гарды и резолверы.

import { Component, inject, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';
import { switchMap } from 'rxjs';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

interface Instrument { symbol: string; tickSize: number; }

@Component({ /* ... */ })
export class InstrumentComponent implements OnInit {
  private readonly route = inject(ActivatedRoute);
  private readonly api = inject(InstrumentService);
  instrument?: Instrument;

  ngOnInit(): void {
    // Reactive — re-fetches when :symbol changes and the component is reused
    this.route.paramMap.pipe(
      switchMap(pm => this.api.getBySymbol(pm.get('symbol')!)),
      takeUntilDestroyed(),
    ).subscribe(i => this.instrument = i);

    // Synchronous snapshot — fine only if this route is never reused
    const symbol = this.route.snapshot.params['symbol'];

    // Resolved data combined with static data
    this.route.data.subscribe(d => console.log(d['quote']));
  }
}

Ловушка снимка: переход с /instruments/EURUSD на /instruments/BTCUSD переиспользует компонент, поэтому snapshot.params остаётся на первом значении. Подписывайтесь на paramMap вместо этого.

Ленивая загрузка

Ленивая загрузка — отложенная загрузка кода маршрута до момента, когда пользователь к нему перейдёт, с помощью динамического import(). Уменьшает начальный бандл и ускоряет первую отрисовку.

На основе модулей (loadChildren)

const routes: Routes = [
  {
    path: 'orders',
    loadChildren: () =>
      import('./feature/feature.module').then(m => m.FeatureModule),
  },
];

Standalone-компонент (loadComponent)

const routes: Routes = [
  {
    path: 'instruments/:symbol',
    loadComponent: () =>
      import('./page.component').then(c => c.PageComponent),
  },
];

Стратегии предзагрузки

Предзагрузка скачивает ленивые чанки в фоновом режиме после загрузки начального приложения, благодаря чему навигация ощущается мгновенной без раздувания первого бандла.

NoPreloading — по умолчанию. Ленивые чанки загружаются только при навигации; ничего не предзагружается.

PreloadAllModules — заранее предзагружает каждый ленивый маршрут в фоне после загрузки приложения.

Пользовательская стратегия — реализуйте PreloadingStrategy, чтобы решать для каждого маршрута, обычно на основе флага в route.data.

СтратегияПредзагружаетЛучше всего для
NoPreloading❌ НичегоМножество редко посещаемых маршрутов
PreloadAllModules✅ ВсёМалые/средние приложения, быстрые сети
Пользовательская✅ Выбранное (по флагу в data)Большие приложения — предзагрузка только ключевых маршрутов

Пример пользовательской стратегии предзагрузки

import { Injectable } from '@angular/core';
import { PreloadingStrategy, Route } from '@angular/router';
import { Observable, of } from 'rxjs';

@Injectable({ providedIn: 'root' })
export class CustomPreloadStrategy implements PreloadingStrategy {
  // Preload only routes flagged with data: { preload: true }
  preload(route: Route, load: () => Observable<unknown>): Observable<unknown> {
    return route.data?.['preload'] ? load() : of(null);
  }
}

// Usage with the route definition
const routes: Routes = [
  {
    path: 'orders',
    data: { preload: true },                 // this one preloads
    loadChildren: () => import('./orders/orders.module').then(m => m.OrdersModule),
  },
  {
    path: 'reports',
    data: { preload: false },                // this one waits
    loadChildren: () => import('./reports/reports.module').then(m => m.ReportsModule),
  },
];

// Register it
provideRouter(routes, withPreloading(CustomPreloadStrategy));

Гарды

Гарды — это функции, которые роутер запускает во время навигации, чтобы разрешить, заблокировать или перенаправить её. Каждая возвращает boolean, UrlTree (переадресацию) или асинхронную обёртку (Promise / Observable) над ними. Начиная с Angular 14+, идиоматичной формой является функциональный гард с использованием inject().

canActivate — решает, разрешена ли навигация к маршруту. Возврат false отменяет навигацию; возврат UrlTree перенаправляет.

canActivateChild — выполняет ту же проверку для дочерних маршрутов перед активацией каждого дочернего.

canDeactivate — решает, разрешена ли навигация прочь от текущего маршрута. Классическое применение — предупреждение о несохранённых изменениях в форме.

canLoad / canMatch — предотвращают скачивание лениво загружаемого модуля. canMatch (новее) также может отклонить маршрут, чтобы роутер попробовал следующий подходящий маршрут.

resolve — не гард как таковой, но выполняется в том же конвейере, чтобы предзагрузить данные до активации компонента, чтобы вид отрисовался с готовыми данными.

Функциональные гарды (Angular 14+)

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';

// Minimal inline guard
export const authGuard: CanActivateFn = () => inject(AuthService).isLoggedIn();

// Guard that redirects via a UrlTree instead of just returning false
export const tradingGuard: CanActivateFn = () => {
  const auth = inject(AuthService);
  const router = inject(Router);
  return auth.isLoggedIn() ? true : router.createUrlTree(['/login']);
};

// Registered on a route
const routes: Routes = [
  { path: 'orders', component: OrderListComponent, canActivate: [authGuard] },
];

canActivateChild

import { CanActivateChildFn } from '@angular/router';

export const childAuthGuard: CanActivateChildFn = () => inject(AuthService).isLoggedIn();

const routes: Routes = [
  {
    path: 'orders',
    canActivateChild: [childAuthGuard],     // applies to every child below
    children: [
      { path: '', component: OrderListComponent },
      { path: ':id', component: OrderDetailComponent },
    ],
  },
];

canDeactivate — предупреждение о несохранённых изменениях

import { CanDeactivateFn } from '@angular/router';

// Contract the guarded component must satisfy
export interface CanComponentDeactivate {
  canDeactivate(): boolean;
}

// Generic functional guard
export const unsavedChangesGuard: CanDeactivateFn<CanComponentDeactivate> =
  (component) => component.canDeactivate();

// Component implementing the contract
@Component({ /* ... */ })
export class OrderFormComponent implements CanComponentDeactivate {
  private readonly form = inject(FormBuilder).group({ symbol: [''], qty: [0] });

  canDeactivate(): boolean {
    // Block navigation if there are unsaved edits
    return this.form.pristine
      ? true
      : confirm('You have unsaved changes. Leave anyway?');
  }
}

const routes: Routes = [
  { path: 'orders/new', component: OrderFormComponent, canDeactivate: [unsavedChangesGuard] },
];

canMatch — гард для ленивой загрузки

import { CanMatchFn } from '@angular/router';

export const adminMatchGuard: CanMatchFn = () => inject(AuthService).isAdmin();

const routes: Routes = [
  {
    path: 'reports',
    canMatch: [adminMatchGuard],   // chunk is not even downloaded if false
    loadChildren: () => import('./reports/reports.module').then(m => m.ReportsModule),
  },
];

resolve — предзагрузка данных

import { ResolveFn } from '@angular/router';

interface Order { id: string; symbol: string; qty: number; }

export const orderResolver: ResolveFn<Order> = (route) => {
  const api = inject(OrderService);
  return api.getById(route.paramMap.get('id')!);
};

const routes: Routes = [
  {
    path: 'orders/:id',
    component: OrderDetailComponent,
    resolve: { order: orderResolver },   // available via route.data['order']
  },
];

Чем полезны резолверы: компонент активируется только после готовности данных, поэтому вы избегаете мерцания пустого состояния и проверок на undefined в шаблоне.

События роутера

Router.events — Observable<Event>, излучающий события жизненного цикла при каждой навигации. Подпишитесь, чтобы управлять спиннерами загрузки, аналитикой или поведением прокрутки. Используйте instanceof (или оператор filter), чтобы реагировать на конкретный тип события.

NavigationStart — навигация началась; обрабатывается новый URL.

RoutesRecognized — URL разобран и сопоставлен с конфигурацией маршрутов; целевое дерево маршрутов известно.

RouteConfigLoadStart — конфигурация лениво загружаемого маршрута вот-вот будет получена (начинается скачивание).

RouteConfigLoadEnd — конфигурация ленивого маршрута завершила скачивание.

NavigationEnd — навигация завершилась успешно; новый маршрут активен. Самое используемое событие (аналитика, прокрутка наверх).

NavigationCancel — навигация была отменена, обычно гардом, вернувшим false, или переадресацией через UrlTree.

NavigationError — навигация завершилась неудачей из-за непредвиденной ошибки (например, резолвер выбросил исключение или ленивый чанк не загрузился).

ActivationEnd — гарды/резолверы маршрута отработали, и маршрут активируется (срабатывает для каждого активируемого маршрута, самый глубокий последним).

import { Component, inject } from '@angular/core';
import {
  Router, NavigationStart, NavigationEnd,
  NavigationCancel, NavigationError, Event as RouterEvent,
} from '@angular/router';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

@Component({ /* ... */ })
export class ShellComponent {
  private readonly router = inject(Router);
  loading = false;

  constructor() {
    this.router.events
      .pipe(takeUntilDestroyed())
      .subscribe((e: RouterEvent) => {
        if (e instanceof NavigationStart) this.loading = true;
        if (e instanceof NavigationEnd) this.loading = false;
        if (e instanceof NavigationCancel) this.loading = false;
        if (e instanceof NavigationError) {
          this.loading = false;
          console.error('Navigation failed:', e.error);
        }
      });
  }
}

BreakpointObserver

BreakpointObserver — сервис Angular CDK (@angular/cdk/layout), который реактивно наблюдает за одной или несколькими CSS media-query брейкпоинтами. observe(...) возвращает Observable<BreakpointState>, который излучает всякий раз, когда отслеживаемый брейкпоинт начинает или перестаёт совпадать — идеально для переключения между мобильным и десктопным макетами (например, сворачивания боковой панели роутера на маленьких экранах).

Один брейкпоинт → поток boolean

import { Component, inject } from '@angular/core';
import { BreakpointObserver, Breakpoints } from '@angular/cdk/layout';
import { map } from 'rxjs';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

@Component({ /* ... */ })
export class LayoutComponent {
  private readonly bp = inject(BreakpointObserver);

  readonly isMobile$ = this.bp.observe([Breakpoints.Handset]).pipe(
    map(result => result.matches),
    takeUntilDestroyed(),
  );
}

Несколько брейкпоинтов → именованный результат

readonly layout$ = this.bp
  .observe(['(max-width: 768px)', '(min-width: 1400px)'])
  .pipe(
    map(result => ({
      isMobile: result.breakpoints['(max-width: 768px)'],
      isWide: result.breakpoints['(min-width: 1400px)'],
    })),
  );

Константы Breakpoints

Breakpoints.Handset — устройства размером с телефон (портретная + альбомная ориентация).

Breakpoints.Tablet — устройства размером с планшет.

Breakpoints.Web — десктоп / большие экраны.

Breakpoints.HandsetPortrait — телефоны только в портретной ориентации.

Breakpoints.XSmall — вьюпорты уже 600px.

Breakpoints.Small — вьюпорты шириной 600px–959px.

Breakpoints.Medium — вьюпорты шириной 960px–1279px.

Breakpoints.Large — вьюпорты шириной 1280px–1919px.

Breakpoints.XLarge — вьюпорты шириной 1920px и больше.

Очистка: пропускайте через takeUntilDestroyed() (или используйте пайп async в шаблоне), чтобы слушатель media-query был удалён при уничтожении компонента.

Вопросы и ответы — собеседование

1. В чём разница между RouterModule.forRoot и forChild?

forRoot вызывается ровно один раз в корневом модуле. Он регистрирует маршруты верхнего уровня и создаёт singleton-сервис Router вместе с глобальными провайдерами (предзагрузка, восстановление прокрутки). forChild используется в функциональных модулях для регистрации дополнительных маршрутов без повторного создания сервиса Router — двойной вызов forRoot создал бы дублирующиеся экземпляры роутера и сломал бы навигацию. В standalone-приложениях provideRouter(routes) заменяет forRoot, а маршруты функций предоставляются напрямую через loadChildren.

2. Когда следует подписываться на paramMap вместо чтения snapshot.params?

Используйте snapshot.params только когда компонент уничтожается и создаётся заново при каждой навигации. Когда вы переходите между двумя экземплярами одного и того же маршрута — например, /instruments/EURUSD → /instruments/BTCUSD — Angular переиспользует компонент, поэтому ngOnInit не запускается снова, и снимок сохраняет старое значение. Подписка на paramMap (Observable) излучает новый symbol и позволяет повторно загрузить данные, обычно через switchMap.

3. В чём разница между canLoad и canMatch?

Оба могут предотвратить скачивание ленивого чанка. canLoad (устаревший) только блокирует загрузку — если он возвращает false, навигация просто завершается неудачей. canMatch (Angular 14.1+) мощнее: он решает, совпадает ли маршрут вообще, поэтому при возврате false роутер двигается дальше и пробует следующий подходящий маршрут. Это даёт паттерны вроде отдачи разных компонентов по одному пути в зависимости от роли или фичефлага. canMatch также работает для немедленно загружаемых (не ленивых) маршрутов, поэтому это рекомендуемая современная замена.

4. Как выполнить переадресацию из гарда?

Верните UrlTree вместо false. Внедрите Router и постройте дерево с помощью router.createUrlTree(['/login']). Возврат UrlTree отменяет текущую навигацию и запускает новую к этому URL — для исходной навигации срабатывает событие NavigationCancel. Возврат простого false лишь блокирует навигацию без переадресации, что обычно оставляет пользователя на пустой или предыдущей странице.

5. Что делает резолвер и в чём его компромисс?

Резолвер (ResolveFn) предзагружает данные до активации маршрута, предоставляя их через route.data, чтобы компонент отрисовался с уже присутствующими данными — без мерцания пустого состояния и проверок на undefined в шаблоне. Компромисс — воспринимаемая задержка: навигация блокируется, пока резолвер не завершится, поэтому медленный API делает весь переход медленным. Для некритичных данных предпочитайте загрузку внутри компонента с состоянием загрузки; резервируйте резолверы для данных, без которых страница действительно не может отрисоваться.

Тест

1. Какой API регистрирует маршруты в standalone начальной загрузке?
provideRouter(routes) передаётся в bootstrapApplication через ApplicationConfig. forRoot/forChild — это API NgModule; forChild для функциональных модулей, а не для начальной загрузки.
2. Какой гард предотвращает навигацию прочь от маршрута с несохранёнными изменениями?
canDeactivate срабатывает при уходе с маршрута — классическое применение это подтверждение перед сбросом несохранённых правок формы. canActivate охраняет вход; canMatch охраняет совпадение/загрузку; resolve предзагружает данные.
3. Какая стратегия предзагрузки используется по умолчанию?
NoPreloading используется по умолчанию — ленивые чанки загружаются только при навигации. PreloadAllModules заранее предзагружает всё; пользовательская PreloadingStrategy может предзагружать выборочно на основе данных маршрута.
4. Какое событие сигнализирует, что навигация завершилась успешно?
NavigationEnd срабатывает, когда новый маршрут активен — используется для аналитики и прокрутки наверх. NavigationStart начинает её, RoutesRecognized сопоставляет URL, NavigationCancel означает, что гард её заблокировал.
