Route GuardsГарды роутера (Guards)

Protecting routes: CanActivate, CanActivateChild, CanDeactivate, CanMatch and Resolve — functional and class-based guards with theory and examples built around a trading platform.Защита маршрутов: CanActivate, CanActivateChild, CanDeactivate, CanMatch и Resolve — функциональные и классовые гарды с теорией и примерами на торговой платформе.

What is a guard

A guard (route guard) is a function or class that the router calls before navigation to decide whether to allow the transition, cancel it, or redirect the user. It is the central place for authorization, permission checks, and preventing the loss of unsaved data.

Types of guards

GuardWhen it runsTypical scenario
CanActivateBefore entering a routeAuthentication required
CanActivateChildBefore entering a child routeProtect an entire section (e.g. /admin/*)
CanDeactivateBefore leaving a routeWarn about an unsaved order form
CanMatchBefore matching a route (and lazy loading)Feature flags, A/B, roles before loading the chunk
ResolveBefore activation — preloading dataLoad the instrument before rendering the page

What a guard can return

Any guard can return a synchronous value or an asynchronous stream — the router will wait for it:

  • boolean — true allows the transition, false cancels it.
  • UrlTree — cancels the transition and performs a redirect to that URL.
  • Observable<boolean | UrlTree> or Promise<...> — for asynchronous checks (HTTP, store state).

Tip: return a UrlTree instead of manually calling router.navigate() inside a guard — this keeps navigation atomic, without a "flash" of the intermediate route.

Modern approach: since Angular 14, functional guards (CanActivateFn, etc.) are preferred — they are simply functions that use inject(). Class-based guards (implements CanActivate) are considered legacy; CanLoad has been replaced by CanMatch.

CanActivate

CanActivate is a guard that denies or allows entry to a route. It returns a boolean or a UrlTree (redirect).

Functional guard (recommended)

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from './auth.service';

export const authGuard: CanActivateFn = (route, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);

  if (auth.isAuthenticated()) {
    return true;
  }
  // Cancel the transition and redirect to /login, preserving the return address
  return router.createUrlTree(['/login'], {
    queryParams: { returnUrl: state.url },
  });
};

Role check (guard factory)

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';

// Parameterized guard: roleGuard('TRADER')
export function roleGuard(required: string): CanActivateFn {
  return () => {
    const auth = inject(AuthService);
    const router = inject(Router);
    return auth.hasRole(required) ? true : router.createUrlTree(['/forbidden']);
  };
}

Asynchronous check via the store

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { map } from 'rxjs';

export const sessionGuard: CanActivateFn = () => {
  const auth = inject(AuthService);
  const router = inject(Router);
  // isAuthenticated$ — Observable<boolean>
  return auth.isAuthenticated$.pipe(
    map(ok => ok ? true : router.createUrlTree(['/login'])),
  );
};

Class-based guard (legacy style)

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private auth: AuthService, private router: Router) {}

  canActivate(): boolean | UrlTree {
    return this.auth.isAuthenticated() ? true : this.router.createUrlTree(['/login']);
  }
}

CanActivateChild

CanActivateChild is the same as CanActivate, but it runs before entering each child route. It is convenient for protecting an entire section of the application with a single guard attached to the parent.

import { CanActivateChildFn } from '@angular/router';

export const adminAreaGuard: CanActivateChildFn = (childRoute, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);
  return auth.hasRole('ADMIN') ? true : router.createUrlTree(['/forbidden']);
};

// In the route config, a guard on the parent protects all nested paths:
const routes: Routes = [
  {
    path: 'admin',
    canActivateChild: [adminAreaGuard],
    children: [
      { path: 'users', component: UsersComponent },
      { path: 'audit', component: AuditComponent },
    ],
  },
];

CanDeactivate

CanDeactivate is a guard that denies or allows leaving a route. The main scenario is to warn the user about unsaved changes (for example, a half-filled order ticket).

import { CanDeactivateFn } from '@angular/router';

// The contract implemented by components with a "dirty" form
export interface CanComponentDeactivate {
  canDeactivate: () => boolean | Observable<boolean>;
}

export const pendingChangesGuard: CanDeactivateFn<CanComponentDeactivate> =
  (component) => component.canDeactivate ? component.canDeactivate() : true;

The component itself decides whether leaving is allowed:

@Component({ /* ... */ })
export class OrderTicketComponent implements CanComponentDeactivate {
  form = inject(FormBuilder).group({ symbol: [''], quantity: [0] });

  canDeactivate(): boolean {
    // If the form is untouched or already saved — leaving is allowed without questions
    if (this.form.pristine) return true;
    return confirm('Заявка не сохранена. Покинуть страницу?');
  }
}

Gotcha: CanDeactivate does not intercept closing the tab or refreshing the page. For that you need a separate beforeunload listener at the browser level.

CanMatch (and the legacy CanLoad)

CanMatch is a guard that decides whether a route matches at all. If it returns false, the router behaves as if this route does not exist and tries the next one. It runs before lazy loading the chunk, so it saves traffic.

import { CanMatchFn, Route } from '@angular/router';

export const featureFlagGuard = (flag: string): CanMatchFn => {
  return () => inject(FeatureFlagsService).isEnabled(flag);
};

const routes: Routes = [
  {
    path: 'analytics',
    canMatch: [featureFlagGuard('advancedAnalytics')],
    loadComponent: () => import('./analytics.component').then(c => c.AnalyticsComponent),
  },
  // Fallback if the feature is off — the same path leads to a stub
  { path: 'analytics', component: ComingSoonComponent },
];

CanMatch vs CanActivate: CanMatch can be used to pick different components for the same path (by role, by flag), and also to avoid loading the lazy module entirely. CanActivate runs only after the route has been matched.

Legacy: CanLoad has been replaced by CanMatch. In new projects, use only CanMatch.

Resolve

Resolve is a data-preloading guard: the router waits until the resolver fetches the data, and only then activates the component. This removes the "flash" of an empty screen and the loading state inside the component.

import { ResolveFn } from '@angular/router';

export const instrumentResolver: ResolveFn<Instrument> = (route) => {
  const service = inject(InstrumentService);
  const symbol = route.paramMap.get('symbol')!;
  return service.getOne(symbol); // Observable<Instrument>
};

const routes: Routes = [
  {
    path: 'instruments/:symbol',
    component: InstrumentComponent,
    resolve: { instrument: instrumentResolver },
  },
];

In the component, the ready data is available from ActivatedRoute:

export class InstrumentComponent {
  private route = inject(ActivatedRoute);
  // The data is already loaded by the resolver — no spinner and no undefined
  instrument = this.route.snapshot.data['instrument'] as Instrument;
}

Gotcha: a heavy resolver delays the entire navigation — the user sees a "freeze". For long-running requests it is better to show the page right away and load the data inside the component (a skeleton), and keep the resolver for fast, critical data.

Execution order

During navigation, the router runs guards in a strict order. If any of them returns false or a UrlTree, navigation is aborted and the rest do not run:

CanDeactivate (уход с текущего маршрута)
   ↓
CanMatch (сопоставление маршрута, до ленивой загрузки)
   ↓
CanActivate (вход на маршрут)
   ↓
CanActivateChild (вход на дочерние маршруты)
   ↓
Resolve (предзагрузка данных)
   ↓
Активация компонента

Guards of the same type on the same route run in the order they are listed in the array (canActivate: [a, b] → a first). If they all return streams, the router waits for them to complete.

Registration in the route config

import { Routes } from '@angular/router';

export const routes: Routes = [
  { path: 'login', component: LoginComponent },
  {
    path: 'orders',
    canActivate: [authGuard, roleGuard('TRADER')],
    canDeactivate: [pendingChangesGuard],
    component: OrdersComponent,
  },
  {
    path: 'instruments/:symbol',
    canActivate: [authGuard],
    resolve: { instrument: instrumentResolver },
    component: InstrumentComponent,
  },
  {
    path: 'admin',
    canMatch: [featureFlagGuard('adminPanel')],
    canActivateChild: [adminAreaGuard],
    loadChildren: () => import('./admin/admin.routes').then(m => m.ADMIN_ROUTES),
  },
];

Questions & Answers

1. What is the difference between CanActivate and CanMatch?

CanMatch runs at the route matching stage — before lazy loading the chunk — and on false the router treats the route as not matching and tries the next one. CanActivate runs only after matching, right before the component is activated. That is why CanMatch is used for feature flags, roles, and selecting different components for the same path, as well as to avoid loading lazy code at all.

2. Why is it better to return a UrlTree rather than call router.navigate() inside a guard?

Returning a UrlTree makes the redirect part of the current navigation — it is cancelled and immediately replaced by a new one, atomically. A manual navigate() starts a second navigation in parallel with cancelling the first, which can cause races and a brief "flash" of the original route.

3. How do you build a guard that protects against losing an unsaved form?

Use CanDeactivate. You declare an interface with a canDeactivate() method that the component implements; the functional guard simply delegates the call to this method. The component returns true if the form is pristine or saved, otherwise it asks for confirmation. Important: closing/refreshing the tab is not caught by this guard — you need beforeunload.

4. What can a guard return and how does the router handle it?

boolean (true — allow, false — cancel), UrlTree (cancel and redirect), or an Observable/Promise of these values for asynchronous checks. The router waits for the stream/promise to complete before continuing navigation.

5. Why do you need Resolve if you can load the data in ngOnInit?

Resolve guarantees that the data is ready before the component renders — there is no intermediate empty/loading state and no undefined in the template. This is convenient for critical data (an instrument card by symbol). The downside is that it delays the entire navigation, so for long-running requests it is better to show the page right away with a skeleton and keep the resolver for fast data.

Quiz

1. Which guard runs before lazy loading a module?
CanMatch is checked at the route matching stage, before loading the chunk, so on false the lazy code is not downloaded.
2. To cancel a transition and immediately redirect the user, a guard must return…
UrlTree cancels the current navigation and atomically replaces it with a redirect. It is created via router.createUrlTree([...]) or parseUrl.
3. Which modern API is preferred for guards in Angular 14+?
Functional guards are plain functions that use inject(). Class-based guards and CanLoad are considered legacy.
4. Which guard is used to warn about unsaved form changes?
CanDeactivate runs when leaving a route and lets the component ask for confirmation if the form is "dirty".

Что такое гард

Гард (route guard) — функция или класс, которые роутер вызывает перед навигацией, чтобы решить, разрешить переход, отменить его или перенаправить пользователя. Это центральная точка для авторизации, проверки прав и предотвращения потери несохранённых данных.

Типы гардов

ГардКогда срабатываетТипичный сценарий
CanActivateПеред входом на маршрутТребуется аутентификация
CanActivateChildПеред входом на дочерний маршрутЗащита всей секции (например, /admin/*)
CanDeactivateПеред уходом с маршрутаПредупреждение о несохранённой форме заявки
CanMatchПеред сопоставлением маршрута (и ленивой загрузкой)Feature-флаги, A/B, роли до загрузки чанка
ResolveПеред активацией — предзагрузка данныхЗагрузить инструмент до отрисовки страницы

Что может вернуть гард

Любой гард может вернуть синхронное значение либо асинхронный поток — роутер дождётся его:

  • boolean — true разрешает переход, false отменяет.
  • UrlTree — отменяет переход и выполняет редирект по этому URL.
  • Observable<boolean | UrlTree> или Promise<...> — для асинхронных проверок (HTTP, состояние стора).

Совет: возвращайте UrlTree вместо ручного вызова router.navigate() внутри гарда — так навигация остаётся атомарной, без «мигания» промежуточного маршрута.

Современный подход: начиная с Angular 14 предпочтительны функциональные гарды (CanActivateFn и т.д.) — это просто функции, использующие inject(). Классовые гарды (implements CanActivate) считаются устаревшими; CanLoad заменён на CanMatch.

CanActivate

CanActivate — гард, который запрещает или разрешает вход на маршрут. Возвращает boolean или UrlTree (редирект).

Функциональный гард (рекомендуется)

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from './auth.service';

export const authGuard: CanActivateFn = (route, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);

  if (auth.isAuthenticated()) {
    return true;
  }
  // Отменяем переход и редиректим на /login, сохраняя адрес возврата
  return router.createUrlTree(['/login'], {
    queryParams: { returnUrl: state.url },
  });
};

Проверка ролей (фабрика гардов)

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';

// Параметризуемый гард: roleGuard('TRADER')
export function roleGuard(required: string): CanActivateFn {
  return () => {
    const auth = inject(AuthService);
    const router = inject(Router);
    return auth.hasRole(required) ? true : router.createUrlTree(['/forbidden']);
  };
}

Асинхронная проверка через стор

import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { map } from 'rxjs';

export const sessionGuard: CanActivateFn = () => {
  const auth = inject(AuthService);
  const router = inject(Router);
  // isAuthenticated$ — Observable<boolean>
  return auth.isAuthenticated$.pipe(
    map(ok => ok ? true : router.createUrlTree(['/login'])),
  );
};

Классовый гард (устаревший стиль)

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private auth: AuthService, private router: Router) {}

  canActivate(): boolean | UrlTree {
    return this.auth.isAuthenticated() ? true : this.router.createUrlTree(['/login']);
  }
}

CanActivateChild

CanActivateChild — то же, что CanActivate, но срабатывает перед входом на каждый дочерний маршрут. Удобно защищать целую секцию приложения одним гардом, навешанным на родителя.

import { CanActivateChildFn } from '@angular/router';

export const adminAreaGuard: CanActivateChildFn = (childRoute, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);
  return auth.hasRole('ADMIN') ? true : router.createUrlTree(['/forbidden']);
};

// В конфигурации роутов гард на родителе защищает все вложенные пути:
const routes: Routes = [
  {
    path: 'admin',
    canActivateChild: [adminAreaGuard],
    children: [
      { path: 'users', component: UsersComponent },
      { path: 'audit', component: AuditComponent },
    ],
  },
];

CanDeactivate

CanDeactivate — гард, который запрещает или разрешает уход с маршрута. Главный сценарий — предупредить пользователя о несохранённых изменениях (например, наполовину заполненная заявка на сделку).

import { CanDeactivateFn } from '@angular/router';

// Контракт, который реализуют компоненты с «грязной» формой
export interface CanComponentDeactivate {
  canDeactivate: () => boolean | Observable<boolean>;
}

export const pendingChangesGuard: CanDeactivateFn<CanComponentDeactivate> =
  (component) => component.canDeactivate ? component.canDeactivate() : true;

Компонент сам решает, можно ли уходить:

@Component({ /* ... */ })
export class OrderTicketComponent implements CanComponentDeactivate {
  form = inject(FormBuilder).group({ symbol: [''], quantity: [0] });

  canDeactivate(): boolean {
    // Если форма не тронута или уже сохранена — уходить можно без вопросов
    if (this.form.pristine) return true;
    return confirm('Заявка не сохранена. Покинуть страницу?');
  }
}

Подводный камень: CanDeactivate не перехватывает закрытие вкладки или обновление страницы. Для этого нужен отдельный слушатель beforeunload на уровне браузера.

CanMatch (и устаревший CanLoad)

CanMatch — гард, который решает, подходит ли маршрут вообще. Если он возвращает false, роутер ведёт себя так, будто этого маршрута нет, и пробует следующий. Срабатывает до ленивой загрузки чанка, поэтому экономит трафик.

import { CanMatchFn, Route } from '@angular/router';

export const featureFlagGuard = (flag: string): CanMatchFn => {
  return () => inject(FeatureFlagsService).isEnabled(flag);
};

const routes: Routes = [
  {
    path: 'analytics',
    canMatch: [featureFlagGuard('advancedAnalytics')],
    loadComponent: () => import('./analytics.component').then(c => c.AnalyticsComponent),
  },
  // Фолбэк, если фича выключена — тот же путь ведёт на заглушку
  { path: 'analytics', component: ComingSoonComponent },
];

CanMatch vs CanActivate: CanMatch можно использовать для выбора разных компонентов на одном пути (по роли, флагу), а также чтобы вообще не грузить ленивый модуль. CanActivate запускается уже после сопоставления маршрута.

Устаревшее: CanLoad заменён на CanMatch. В новых проектах используйте только CanMatch.

Resolve

Resolve — гард предзагрузки данных: роутер ждёт, пока резолвер получит данные, и только потом активирует компонент. Это убирает «мигание» пустого экрана и состояние загрузки внутри компонента.

import { ResolveFn } from '@angular/router';

export const instrumentResolver: ResolveFn<Instrument> = (route) => {
  const service = inject(InstrumentService);
  const symbol = route.paramMap.get('symbol')!;
  return service.getOne(symbol); // Observable<Instrument>
};

const routes: Routes = [
  {
    path: 'instruments/:symbol',
    component: InstrumentComponent,
    resolve: { instrument: instrumentResolver },
  },
];

В компоненте готовые данные доступны из ActivatedRoute:

export class InstrumentComponent {
  private route = inject(ActivatedRoute);
  // Данные уже загружены резолвером — без спиннера и без undefined
  instrument = this.route.snapshot.data['instrument'] as Instrument;
}

Подводный камень: тяжёлый резолвер задерживает всю навигацию — пользователь видит «зависание». Для долгих запросов лучше показать страницу сразу и грузить данные внутри компонента (скелетон), а резолвер оставить для быстрых критичных данных.

Порядок выполнения

При навигации роутер выполняет гарды в строгом порядке. Если любой из них вернёт false или UrlTree, навигация прерывается, а остальные не запускаются:

CanDeactivate (уход с текущего маршрута)
   ↓
CanMatch (сопоставление маршрута, до ленивой загрузки)
   ↓
CanActivate (вход на маршрут)
   ↓
CanActivateChild (вход на дочерние маршруты)
   ↓
Resolve (предзагрузка данных)
   ↓
Активация компонента

Гарды одного типа на одном маршруте выполняются в порядке их перечисления в массиве (canActivate: [a, b] → сначала a). Если все возвращают потоки, роутер ждёт их завершения.

Регистрация в конфигурации роутов

import { Routes } from '@angular/router';

export const routes: Routes = [
  { path: 'login', component: LoginComponent },
  {
    path: 'orders',
    canActivate: [authGuard, roleGuard('TRADER')],
    canDeactivate: [pendingChangesGuard],
    component: OrdersComponent,
  },
  {
    path: 'instruments/:symbol',
    canActivate: [authGuard],
    resolve: { instrument: instrumentResolver },
    component: InstrumentComponent,
  },
  {
    path: 'admin',
    canMatch: [featureFlagGuard('adminPanel')],
    canActivateChild: [adminAreaGuard],
    loadChildren: () => import('./admin/admin.routes').then(m => m.ADMIN_ROUTES),
  },
];

Вопросы и ответы

1. В чём разница между CanActivate и CanMatch?

CanMatch срабатывает на этапе сопоставления маршрута — до ленивой загрузки чанка — и при false роутер считает, что маршрут не подходит, и пробует следующий. CanActivate запускается уже после сопоставления, перед активацией компонента. Поэтому CanMatch используют для feature-флагов, ролей и выбора разных компонентов на одном пути, а также чтобы вообще не грузить ленивый код.

2. Почему лучше вернуть UrlTree, а не вызвать router.navigate() внутри гарда?

Возврат UrlTree делает редирект частью текущей навигации — она отменяется и сразу заменяется новой, атомарно. Ручной navigate() запускает вторую навигацию параллельно с отменой первой, что может вызвать гонки и кратковременное «мигание» исходного маршрута.

3. Как сделать гард, защищающий от потери несохранённой формы?

Использовать CanDeactivate. Объявляют интерфейс с методом canDeactivate(), который реализует компонент; функциональный гард просто делегирует вызов этому методу. Компонент возвращает true, если форма pristine или сохранена, иначе спрашивает подтверждение. Важно: закрытие/обновление вкладки этим гардом не ловится — нужен beforeunload.

4. Что может возвращать гард и как роутер это обрабатывает?

boolean (true — разрешить, false — отменить), UrlTree (отменить и сделать редирект), либо Observable/Promise этих значений для асинхронных проверок. Роутер дожидается завершения потока/промиса, прежде чем продолжить навигацию.

5. Зачем нужен Resolve, если можно загрузить данные в ngOnInit?

Resolve гарантирует, что данные готовы до отрисовки компонента — нет промежуточного пустого/загрузочного состояния и нет undefined в шаблоне. Это удобно для критичных данных (карточка инструмента по символу). Минус — задержка всей навигации, поэтому для долгих запросов лучше показывать страницу сразу со скелетоном, а резолвер оставлять для быстрых данных.

Тест

1. Какой гард срабатывает до ленивой загрузки модуля?
CanMatch проверяется на этапе сопоставления маршрута, до загрузки чанка, поэтому при false ленивый код не скачивается.
2. Чтобы отменить переход и сразу перенаправить пользователя, гард должен вернуть…
UrlTree отменяет текущую навигацию и атомарно заменяет её редиректом. Создаётся через router.createUrlTree([...]) или parseUrl.
3. Какой современный API предпочтителен для гардов в Angular 14+?
Функциональные гарды — это простые функции, использующие inject(). Классовые гарды и CanLoad считаются устаревшими.
4. Какой гард используют, чтобы предупредить о несохранённых изменениях формы?
CanDeactivate срабатывает при уходе с маршрута и позволяет компоненту запросить подтверждение, если форма «грязная».
