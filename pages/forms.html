Формы в Angular

Две стратегии работы с формами, которые разбирают на собеседованиях для senior: template-driven и reactive. Подробный разбор реактивных форм — FormControl, FormGroup, FormArray, FormBuilder, встроенные и кастомные валидаторы, межполевая валидация и динамические формы.

Template-driven против Reactive

Angular предлагает два способа построения форм. Оба пригодны для продакшена, но они по-разному определяют, где находится источник истины: в шаблоне (template-driven) или в классе компонента (reactive).

Template-driven формы — модель формы строится неявно директивами (ngModel, ngForm), размещёнными в шаблоне. Angular выводит структуру из DOM. Подходят для простых, преимущественно статичных форм.

Reactive формы — модель формы задаётся явно в классе как дерево объектов FormControl/FormGroup/FormArray. Шаблон привязывается к этой модели. Подходят для сложных, динамических и тщательно валидируемых форм.

НастройкаНеявно через ngModelЯвный FormGroup в классе

Пример template-driven

<form #f="ngForm" (ngSubmit)="submit(f.value)">
  <input name="symbol" ngModel required />
  <input name="quantity" type="number" ngModel min="1" />
  <button [disabled]="f.invalid">Place</button>
</form>

Требует FormsModule. Модель живёт в локальной для шаблона ссылке #f.

Пример reactive

import { Component } from '@angular/core';
import { FormControl, FormGroup, Validators, ReactiveFormsModule } from '@angular/forms';

@Component({
  selector: 'app-ticket',
  standalone: true,
  imports: [ReactiveFormsModule],
  templateUrl: './ticket.component.html',
})
export class TicketComponent {
  readonly form = new FormGroup({
    symbol: new FormControl('', { nonNullable: true, validators: [Validators.required] }),
    quantity: new FormControl(0, { nonNullable: true, validators: [Validators.min(1)] }),
  });
}

Требует ReactiveFormsModule. Модель — это явное поле form.

Правило для senior: в FinTech-приложении — ордерные тикеты, многоногие стратегии, динамические сетки позиций — выбирайте reactive формы. Template-driven годится для формы входа или фильтра с одним полем.

Reactive формы — строительные блоки

Реактивные формы — это дерево из трёх классов контролов, все наследуются от AbstractControl. Знание точной роли каждого — гарантированный вопрос на собеседовании.

FormControl — отслеживает значение и валидность одного поля (один input, чекбокс или select). Атомарная единица реактивной формы.

FormGroup — группа контролов по именам, представляющая своё значение как единый объект. Валидность — это логическое И его потомков. Моделирует форму фиксированной формы.

FormArray — упорядоченный список контролов, представляющий своё значение как массив. Контролы можно добавлять/удалять во время выполнения — основа динамических полей.

FormBuilder — внедряемый сервис-сокращение, который конструирует вышеперечисленное с меньшим количеством шаблонного кода (fb.group(...), fb.array(...), fb.control(...)).

AbstractControl — общий базовый класс для всех трёх. Кастомные валидаторы получают AbstractControl, поэтому работают на любом уровне дерева.

import { FormControl, FormGroup, FormArray } from '@angular/forms';

// Single field
const symbol = new FormControl<string>('EURUSD', { nonNullable: true });

// A fixed-shape group → value is an object
const order = new FormGroup({
  symbol: new FormControl('EURUSD', { nonNullable: true }),
  quantity: new FormControl(100, { nonNullable: true }),
});
// order.value === { symbol: 'EURUSD', quantity: 100 }

// A dynamic list → value is an array
const legs = new FormArray([
  new FormControl('EURUSD', { nonNullable: true }),
  new FormControl('GBPUSD', { nonNullable: true }),
]);
// legs.value === ['EURUSD', 'GBPUSD']

FormBuilder — сокращённая запись

Внедрение FormBuilder (или типизированного NonNullableFormBuilder) убирает повторяющийся шум вида new FormControl(...).

import { inject } from '@angular/core';
import { FormBuilder, Validators } from '@angular/forms';

const fb = inject(FormBuilder);

const form = fb.group({
  symbol: ['EURUSD', [Validators.required]],
  quantity: [100, [Validators.required, Validators.min(1)]],
});

Предпочитайте fb.nonNullable.group(...), чтобы reset() восстанавливал исходное значение вместо null, а выводимые типы были non-nullable.

Чтение и запись значений

Четыре метода покрывают почти любое взаимодействие со значением контрола или группы. Путаница между ними — классический баг.

setValue(obj) — устанавливает ВСЕ поля сразу. Нужно передать полный объект, точно соответствующий форме — отсутствующий или лишний ключ выбросит ошибку.

patchValue(partial) — устанавливает НЕКОТОРЫЕ поля. Неуказанные ключи остаются без изменений; неизвестные ключи молча игнорируются. Безопаснее всего для частичных обновлений (например, предзаполнение с сервера).

getRawValue() — возвращает значение, включая отключённые контролы. Обычное .value пропускает отключённые контролы — частый источник багов «моё поле пропало».

reset(value?) — сбрасывает значение (к переданному значению или к исходному), очищает dirty/touched и заново запускает валидацию.

// setValue — ALL fields required, complete object
form.setValue({ symbol: 'GBPUSD', quantity: 250 });

// patchValue — SOME fields, the rest untouched
form.patchValue({ quantity: 250 });

// disabled controls are excluded from .value but kept by getRawValue()
form.controls.symbol.disable();
form.value;          // { quantity: 250 }            ← symbol missing
form.getRawValue();  // { symbol: 'GBPUSD', quantity: 250 }

// reset back to the initial (non-nullable) values
form.reset();

Ловушка: вызов setValue с частичным объектом выбрасывает Must supply a value for form control with name: '...'. Для частичных данных используйте patchValue.

Встроенные валидаторы

Валидаторы — это чистые функции типа (control: AbstractControl) => ValidationErrors | null. Возврат null означает «валидно»; возврат объекта ошибок означает «невалидно». Angular поставляет их из коробки в классе Validators.

Validators.required — значение должно быть непустым (не null/undefined/''/пустой массив). Ключ ошибки: required.

Validators.minLength(n) — длина строки/массива ≥ n. Ключ ошибки: minlength.

Validators.maxLength(n) — длина строки/массива ≤ n. Ключ ошибки: maxlength.

Validators.min(n) — числовое значение ≥ n. Ключ ошибки: min.

Validators.max(n) — числовое значение ≤ n. Ключ ошибки: max.

Validators.email — значение соответствует шаблону email. Ключ ошибки: email.

Validators.pattern(regex) — значение соответствует заданному RegExp или строковому шаблону. Ключ ошибки: pattern.

Validators.requiredTrue — значение должно быть строго true (для чекбоксов «Я принимаю условия»). Ключ ошибки: required.

Validators.nullValidator — пустышка, всегда возвращающая null. Полезна как заглушка/значение по умолчанию.

import { FormControl, Validators } from '@angular/forms';

const quantity = new FormControl(0, {
  nonNullable: true,
  validators: [Validators.required, Validators.min(1), Validators.max(1_000_000)],
});

const symbol = new FormControl('', {
  nonNullable: true,
  validators: [Validators.required, Validators.pattern(/^[A-Z]{6}$/)],
});

quantity.setValue(0);
quantity.errors; // { min: { min: 1, actual: 0 } }
quantity.valid;  // false

Проверяйте control.errors в шаблоне для отрисовки сообщений: @if (form.controls.quantity.hasError('min')) { ... }.

Кастомные валидаторы

Синхронный валидатор — функция (control: AbstractControl) => ValidationErrors | null, которая выполняется немедленно при каждом изменении значения.

Асинхронный валидатор — функция, возвращающая Promise или Observable с ValidationErrors | null, используется когда валидация требует обращения к серверу. Пока она выполняется, статус контрола — PENDING.

Синхронный пример — положительное число

import { AbstractControl, ValidationErrors } from '@angular/forms';

export function positiveNumber(control: AbstractControl): ValidationErrors | null {
  return control.value > 0 ? null : { notPositive: true };
}

// usage
const price = new FormControl(0, { nonNullable: true, validators: [positiveNumber] });
price.errors; // { notPositive: true }

Асинхронный пример — уникальный email

Фабрика async-валидатора захватывает сервис, ничего здесь не дебаунсит (тайминг берёт на себя планировщик формы) и преобразует ответ сервера в объект ошибок. Всегда делайте catchError в of(null), чтобы сбой сети не пометил поле невалидным навсегда.

import { AbstractControl, AsyncValidatorFn, ValidationErrors } from '@angular/forms';
import { Observable, of } from 'rxjs';
import { map, catchError } from 'rxjs/operators';

interface AccountService {
  checkEmail(email: string): Observable<boolean>; // true === already taken
}

export function uniqueEmail(service: AccountService): AsyncValidatorFn {
  return (control: AbstractControl): Observable<ValidationErrors | null> =>
    service.checkEmail(control.value).pipe(
      map(taken => (taken ? { emailTaken: true } : null)),
      catchError(() => of(null)),
    );
}

// usage — async validators go in the 3rd position / asyncValidators option
const email = new FormControl('', {
  nonNullable: true,
  validators: [Validators.required, Validators.email],
  asyncValidators: [uniqueEmail(accountService)],
});

Пока async-валидатор выполняется, control.status === 'PENDING', а control.valid равно false — блокируйте кнопку отправки по form.pending || form.invalid.

valueChanges и statusChanges

Каждый AbstractControl предоставляет два реактивных потока. Это мост между формами и RxJS.

valueChanges — Observable, который эмитит новое значение контрола при каждом изменении значения (пользовательский ввод или программный, если не передано { emitEvent: false }).

statusChanges — Observable, который эмитит статус контрола при каждом его изменении: 'VALID', 'INVALID', 'PENDING' или 'DISABLED'.

import { debounceTime, distinctUntilChanged, switchMap } from 'rxjs/operators';

// React to the symbol field → fetch a live quote
this.form.controls.symbol.valueChanges
  .pipe(
    debounceTime(200),
    distinctUntilChanged(),
    switchMap((symbol: string) => this.quotes.stream(symbol)),
    takeUntilDestroyed(this.destroyRef),
  )
  .subscribe((quote: Quote) => this.lastQuote.set(quote));

// React to overall form status
this.form.statusChanges.subscribe((status) => {
  this.canSubmit.set(status === 'VALID');
});

Всегда отписывайтесь — используйте takeUntilDestroyed() или пайп async. Утёкшая подписка на valueChanges удерживает компонент живым и срабатывает после навигации.

Межполевая валидация

Межполевой (групповой) валидатор — валидатор, прикреплённый к FormGroup, а не к одному контролу, чтобы он мог сравнивать соседние контролы (например, пароль и подтверждение пароля или стоп-лосс и лимитную цену).

Поскольку он живёт на группе, он получает группу как свой AbstractControl и читает потомков через control.get('name'). Ошибка сообщается на группе, а не на отдельных полях.

import { AbstractControl, ValidationErrors, FormGroup, FormControl, Validators } from '@angular/forms';

// Group validator: the two passwords must match
export function passwordsMatch(group: AbstractControl): ValidationErrors | null {
  const password = group.get('password')?.value;
  const confirm = group.get('confirm')?.value;
  return password === confirm ? null : { passwordMismatch: true };
}

const credentials = new FormGroup(
  {
    password: new FormControl('', { nonNullable: true, validators: [Validators.required, Validators.minLength(8)] }),
    confirm: new FormControl('', { nonNullable: true, validators: [Validators.required] }),
  },
  { validators: [passwordsMatch] }, // ← attached to the GROUP
);

credentials.setValue({ password: 'hunter22!', confirm: 'typo' });
credentials.errors; // { passwordMismatch: true }   (on the group)
<!-- read the group-level error in the template -->
@if (credentials.hasError('passwordMismatch') && credentials.get('confirm')?.touched) {
  <p class="error">Passwords do not match</p>
}

Динамические формы с FormArray

FormArray позволяет добавлять и удалять контролы во время выполнения — именно то, что нужно для многоногого ордера или редактируемого списка позиций. Вы изменяете массив через его методы (push, removeAt, insert, clear), а Angular держит представление в синхронизации.

FormArray.push(control) — добавляет контрол в конец.

FormArray.removeAt(index) — удаляет контрол по заданному индексу.

FormArray.at(index) — возвращает контрол по индексу для привязки.

FormArray.clear() — удаляет все контролы.

import { inject } from '@angular/core';
import { FormArray, FormGroup, FormControl, FormBuilder, Validators } from '@angular/forms';

interface OrderLine {
  symbol: string;
  quantity: number;
}

export class BasketComponent {
  private readonly fb = inject(FormBuilder);

  readonly form = this.fb.nonNullable.group({
    account: ['ACC-001', [Validators.required]],
    lines: this.fb.array<FormGroup>([]), // dynamic list of order lines
  });

  get lines(): FormArray {
    return this.form.controls.lines;
  }

  private createLine(line?: Partial<OrderLine>): FormGroup {
    return this.fb.nonNullable.group({
      symbol: [line?.symbol ?? '', [Validators.required, Validators.pattern(/^[A-Z]{6}$/)]],
      quantity: [line?.quantity ?? 1, [Validators.required, Validators.min(1)]],
    });
  }

  addOrderLine(line?: Partial<OrderLine>): void {
    this.lines.push(this.createLine(line));
  }

  removeOrderLine(index: number): void {
    this.lines.removeAt(index);
  }
}
<form [formGroup]="form">
  <input formControlName="account" />

  <div formArrayName="lines">
    @for (line of lines.controls; track $index) {
      <div [formGroupName]="$index">
        <input formControlName="symbol" placeholder="EURUSD" />
        <input formControlName="quantity" type="number" />
        <button type="button" (click)="removeOrderLine($index)">✕</button>
      </div>
    }
  </div>

  <button type="button" (click)="addOrderLine()">+ Add line</button>
</form>

Ловушка: привязывайте каждую строку через [formGroupName]="$index" (число), а не строку. Забытая обёртка formArrayName — причина №1, по которой динамические строки не регистрируются.

Полный пример — ордерный тикет

Реалистичная FinTech-реактивная форма: одноногий ордерный тикет с symbol, side, quantity и price, построенный с FormBuilder, типизированный против доменных моделей и подключённый к живому потоку статуса. Без any.

// ---- domain models ----
type Side = 'BUY' | 'SELL';

interface Instrument {
  symbol: string;
  tickSize: number;
}

interface Quote {
  symbol: string;
  bid: number;
  ask: number;
}

interface Order {
  symbol: string;
  side: Side;
  quantity: number;
  price: number;
}

interface Position {
  symbol: string;
  netQuantity: number;
  avgPrice: number;
}
import { Component, inject, signal } from '@angular/core';
import { FormBuilder, ReactiveFormsModule, Validators } from '@angular/forms';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
import { DestroyRef } from '@angular/core';

@Component({
  selector: 'app-order-ticket',
  standalone: true,
  imports: [ReactiveFormsModule],
  templateUrl: './order-ticket.component.html',
})
export class OrderTicketComponent {
  private readonly fb = inject(FormBuilder);
  private readonly destroyRef = inject(DestroyRef);

  readonly submitting = signal(false);

  readonly form = this.fb.nonNullable.group({
    symbol: ['', [Validators.required, Validators.pattern(/^[A-Z]{6}$/)]],
    side: ['BUY' as Side, [Validators.required]],
    quantity: [1, [Validators.required, Validators.min(1), Validators.max(1_000_000)]],
    price: [0, [Validators.required, Validators.min(0.0001)]],
  });

  constructor() {
    // Disable submit while async checks run or the form is invalid
    this.form.statusChanges
      .pipe(takeUntilDestroyed(this.destroyRef))
      .subscribe((status) => console.log('status:', status));
  }

  submit(): void {
    if (this.form.invalid) {
      this.form.markAllAsTouched();
      return;
    }
    // getRawValue → includes disabled fields, fully typed as Order
    const order: Order = this.form.getRawValue();
    this.submitting.set(true);
    // this.orders.place(order)...
  }

  reset(): void {
    this.form.reset(); // restores '', 'BUY', 1, 0 thanks to nonNullable
  }
}
<form [formGroup]="form" (ngSubmit)="submit()">
  <label>
    Symbol
    <input formControlName="symbol" placeholder="EURUSD" />
  </label>
  @if (form.controls.symbol.hasError('pattern') && form.controls.symbol.touched) {
    <p class="error">Symbol must be 6 uppercase letters</p>
  }

  <label>
    Side
    <select formControlName="side">
      <option value="BUY">Buy</option>
      <option value="SELL">Sell</option>
    </select>
  </label>

  <label>Quantity <input type="number" formControlName="quantity" /></label>
  <label>Price <input type="number" formControlName="price" /></label>

  <button type="submit" [disabled]="form.invalid || submitting()">Place order</button>
  <button type="button" (click)="reset()">Reset</button>
</form>

Почему это уровень собеседования: типизированная non-nullable форма, getRawValue() возвращает полностью типизированный Order, statusChanges управляет кнопкой отправки, markAllAsTouched() показывает ошибки при отправке, а подписка снимается через takeUntilDestroyed.

Вопросы и ответы — вопросы с собеседований

1. Когда вы выберете reactive формы вместо template-driven?

Reactive формы, когда форма сложная, динамическая или тщательно валидируемая — модель явно живёт в классе, синхронна, легко тестируется юнит-тестами на чистом TS и масштабируется до динамических полей FormArray и межполевых валидаторов. Template-driven годится для простых, преимущественно статичных форм (вход, единственный фильтр), где неявной модели ngModel достаточно, но для теста ей нужны DOM и async-тик. В FinTech-приложении — ордерные тикеты, многоногие корзины — вы почти всегда выбираете reactive.

2. В чём разница между setValue и patchValue?

setValue(obj) требует полный объект, покрывающий все контролы — отсутствующий или неизвестный ключ выбросит ошибку. patchValue(partial) обновляет только те ключи, что вы передали, оставляет остальное без изменений и молча игнорирует неизвестные ключи. Используйте setValue, когда у вас есть вся модель и нужна безопасность строгой проверки формы; используйте patchValue для частичных обновлений, таких как предзаполнение с сервера.

3. Почему в form.value может отсутствовать поле и как его вернуть?

Отключённые контролы исключаются из form.value. Используйте form.getRawValue(), чтобы получить значение, включая отключённые контролы. На этом спотыкаются, когда поле условно отключается и затем «исчезает» из отправляемого payload.

4. Как валидировать, что два поля совпадают (например, пароль / подтверждение)?

Прикрепите групповой валидатор к FormGroup, а не к одному контролу. Валидатор получает группу как AbstractControl, читает соседей через group.get('password') и group.get('confirm') и возвращает { passwordMismatch: true } или null. Ошибка сообщается на группе, поэтому шаблон читает group.hasError('passwordMismatch').

5. Как async-валидаторы влияют на статус контрола и в чём подводный камень?

Пока Observable/Promise async-валидатора выполняется, статус контрола — PENDING, а valid равно false, поэтому отправку следует блокировать по form.pending || form.invalid. Главный подводный камень — необработка ошибок: брошенная HTTP-ошибка оставляет поле застрявшим. Всегда делайте catchError(() => of(null)), чтобы сбой сети разрешался в «валидно», а не блокировал форму навсегда.

Тест

1. Какой метод требует полный объект, покрывающий все контролы формы?
setValue() выбрасывает ошибку, если в объекте отсутствует какой-либо контрол. patchValue() обновляет только переданные ключи и игнорирует неизвестные.
2. Контрол отключён. Какой вызов всё равно вернёт его значение?
form.value пропускает отключённые контролы; getRawValue() их включает. Это распространённый баг «моё поле пропало из payload».
3. Куда прикрепить валидатор, сравнивающий два соседних контрола?
Межполевая валидация живёт на FormGroup, чтобы валидатор мог читать обоих соседей через group.get(...). Ошибка сообщается на группе.
4. Пока async-валидатор выполняется, каков статус контрола?
Асинхронная валидация устанавливает статус PENDING (а valid равно false), пока Observable/Promise не завершится. Блокируйте отправку по form.pending || form.invalid.
