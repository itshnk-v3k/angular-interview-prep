Live Coding — задачи и решения

Четырнадцать задач для тестовых заданий и собеседований у доски, которые реально используются на собеседованиях для senior-разработчиков: утилиты с нуля (debounce, throttle, глубокое клонирование), строительные блоки Angular (пайпы, валидаторы, виртуальный скролл, интерцепторы) и классические алгоритмические разминки — каждая с подсказками, решением с комментариями, оценкой сложности и уточняющими вопросами, которые интервьюер задаёт следом.

1. Реализуйте debounce

Задача. «Реализуйте функцию debounce с нуля — без lodash. Она принимает функцию и время ожидания и возвращает новую функцию, которая откладывает вызов оригинала до тех пор, пока с момента последнего вызова не пройдёт wait мс. Мы используем её, чтобы ограничить частоту запросов из поля поиска тикеров, которое бьёт по нашему API котировок на каждое нажатие клавиши».

Вам нужно сохранять ссылку на таймер между вызовами — а это значит замыкание над переменной timer. Каждый вызов сбрасывает предыдущий таймер и планирует новый.

Сохраняйте this и аргументы. Используйте обычную (не стрелочную) возвращаемую функцию, чтобы она наследовала this вызывающего кода, и захватывайте ...args для отложенного вызова.

// Generic, fully typed. T is the wrapped function's signature.
function debounce<T extends (...args: never[]) => void>(
  fn: T,
  wait: number,
): (...args: Parameters<T>) => void {
  // `timer` is captured by the closure and survives between calls.
  let timer: ReturnType<typeof setTimeout> | undefined;

  // Regular function so `this` is whatever the caller binds it to.
  return function (this: unknown, ...args: Parameters<T>): void {
    // Cancel any pending invocation — only the LAST call wins.
    clearTimeout(timer);
    timer = setTimeout(() => {
      // Re-apply the original `this` and arguments.
      fn.apply(this, args);
    }, wait);
  };
}

// Usage: search the quotes API at most once per 300ms of quiet typing.
const search = debounce((term: string) => loadQuotes(term), 300);
input.addEventListener('input', e => search((e.target as HTMLInputElement).value));

Сложность по времени. O(1) на вызов — один clearTimeout/setTimeout. Память O(1) (одна ссылка на таймер).

Уточнение — leading edge, cancel и flush. Интервьюеры почти всегда просят расширить реализацию. Вот версия production-уровня.

interface DebounceOptions { leading?: boolean; trailing?: boolean; }
interface Debounced<T extends (...a: never[]) => void> {
  (...args: Parameters<T>): void;
  cancel(): void;   // drop the pending call
  flush(): void;    // run the pending call right now
}

function debounce<T extends (...a: never[]) => void>(
  fn: T,
  wait: number,
  { leading = false, trailing = true }: DebounceOptions = {},
): Debounced<T> {
  let timer: ReturnType<typeof setTimeout> | undefined;
  let lastArgs: Parameters<T> | undefined;
  let lastThis: unknown;

  const invoke = () => {
    if (lastArgs) fn.apply(lastThis, lastArgs);
    lastArgs = undefined;
  };

  const debounced = function (this: unknown, ...args: Parameters<T>) {
    lastArgs = args;
    lastThis = this;
    const callNow = leading && timer === undefined; // first edge of a burst
    clearTimeout(timer);
    timer = setTimeout(() => {
      timer = undefined;
      if (trailing && !callNow) invoke(); // avoid double-firing leading+trailing
    }, wait);
    if (callNow) invoke();
  } as Debounced<T>;

  debounced.cancel = () => { clearTimeout(timer); timer = undefined; lastArgs = undefined; };
  debounced.flush  = () => { if (timer) { clearTimeout(timer); timer = undefined; invoke(); } };
  return debounced;
}

2. Реализуйте throttle

Задача. «Теперь реализуйте throttle. В отличие от debounce, он гарантирует, что функция выполняется не чаще одного раза в wait мс при непрерывном потоке вызовов — мы используем его для перерисовки живого стакана заявок при потоке тиков из WebSocket. Поддержите leading и trailing edge».

Отслеживайте временную метку последнего вызова. На каждом вызове вычисляйте оставшееся время: если оно ≤ 0, срабатывайте сразу; иначе планируйте trailing-вызов на остаток.

Храните последние аргументы в переменной замыкания, чтобы trailing-вызов использовал самые свежие данные, а не аргументы момента установки таймера.

interface ThrottleOptions { leading?: boolean; trailing?: boolean; }

function throttle<T extends (...a: never[]) => void>(
  fn: T,
  wait: number,
  { leading = true, trailing = true }: ThrottleOptions = {},
): (...args: Parameters<T>) => void {
  let last = 0;                                  // timestamp of last run
  let timer: ReturnType<typeof setTimeout> | undefined;
  let lastArgs: Parameters<T> | undefined;
  let lastThis: unknown;

  const run = (now: number) => {
    last = now;
    fn.apply(lastThis, lastArgs as Parameters<T>);
    lastArgs = undefined;
  };

  return function (this: unknown, ...args: Parameters<T>): void {
    const now = Date.now();
    // If leading is off, treat the very first call as if it just ran.
    if (last === 0 && !leading) last = now;
    const remaining = wait - (now - last);
    lastArgs = args;
    lastThis = this;

    if (remaining <= 0) {
      // Enough time has passed — fire on the leading edge.
      clearTimeout(timer);
      timer = undefined;
      run(now);
    } else if (trailing && timer === undefined) {
      // Schedule a single trailing call for the end of the window.
      timer = setTimeout(() => {
        timer = undefined;
        run(leading ? Date.now() : 0);
      }, remaining);
    }
  };
}

// Repaint the book at most 10x/sec no matter how fast ticks arrive.
const repaint = throttle((q: Quote) => renderBook(q), 100);
socket.onQuote = repaint;

Сложность по времени. O(1) на вызов. Память O(1).

Debounce vs throttle в одну строку: debounce ждёт тишины, затем срабатывает один раз; throttle срабатывает с фиксированной частотой во время активности. Поле поиска → debounce; обработчик scroll/resize/тиков → throttle.

3. Исправьте утечку памяти

Задача. «Этот компонент подписывается на живой поток цен. Он течёт — после нескольких переходов туда и обратно приложение тормозит, а в интерфейсе мелькают устаревшие цены. Найдите баг и исправьте его. Покажите минимум два корректных способа исправления».

Сломанный код:

@Component({
  selector: 'app-ticker',
  template: `<span>{{ price }}</span>`,
})
export class TickerComponent implements OnInit {
  price = 0;

  constructor(private quotes: QuotesService) {}

  ngOnInit(): void {
    // 🐛 LEAK: this subscription is never unsubscribed.
    // Each time the component is created a new subscription is opened;
    // the old ones keep running forever, holding `this` alive (the whole
    // component, its DOM and closures can never be garbage-collected).
    this.quotes.price$.subscribe(p => {
      this.price = p;
    });
  }
}

Подписка переживает компонент, потому что при уничтожении ничто её не закрывает. Либо завершайте её при destroy, либо доверьте управление ею фреймворку.

Самое чистое современное решение вообще убирает ручную подписку — пайп async в шаблоне подписывается и отписывается за вас.

Решение 1 — takeUntilDestroyed() (Angular 16+, современная идиома):

import { Component, inject } from '@angular/core';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

@Component({ selector: 'app-ticker', template: `<span>{{ price }}</span>` })
export class TickerComponent {
  price = 0;
  private quotes = inject(QuotesService);

  constructor() {
    // Must run in an injection context (constructor / field initializer).
    // The stream is auto-completed when the component is destroyed.
    this.quotes.price$
      .pipe(takeUntilDestroyed())
      .subscribe(p => (this.price = p));
  }
}

Решение 2 — пайп async (совсем без ручной подписки — предпочтительно):

@Component({
  selector: 'app-ticker',
  // async pipe subscribes on init and unsubscribes on destroy automatically.
  template: `<span>{{ price$ | async }}</span>`,
})
export class TickerComponent {
  // Just expose the observable — the template does the rest.
  readonly price$ = inject(QuotesService).price$;
}

Решение 3 — классический takeUntil(destroy$) (до 16-й версии, всё ещё часто встречается в легаси-коде):

private destroy$ = new Subject<void>();

ngOnInit(): void {
  this.quotes.price$
    .pipe(takeUntil(this.destroy$))
    .subscribe(p => (this.price = p));
}
ngOnDestroy(): void {
  this.destroy$.next();      // emit → all takeUntil-ed streams complete
  this.destroy$.complete();  // release the Subject itself
}

4. Виртуальный скролл для таблицы на 10 тыс. строк

Задача. «Мы рендерим таблицу из 10 000+ инструментов, и при скролле она сильно лагает. Сделайте её плавной. Объясните, почему ваш подход работает».

Вам не нужны 10 000 DOM-узлов — пользователь видит лишь ~20 за раз. Рендерьте только видимое окно и переиспользуйте узлы по мере прокрутки. В Angular CDK это уже встроено.

Используйте <cdk-virtual-scroll-viewport> с фиксированным itemSize и *cdkVirtualFor вместо *ngFor. Не забудьте trackBy.

import { Component, inject } from '@angular/core';
import { ScrollingModule } from '@angular/cdk/scrolling';
import { AsyncPipe } from '@angular/common';

@Component({
  selector: 'app-instrument-table',
  standalone: true,
  imports: [ScrollingModule, AsyncPipe],
  // itemSize = fixed row height in px. The viewport uses it to compute
  // how many rows fit and what padding to add for the scrollbar.
  template: `
    <cdk-virtual-scroll-viewport itemSize="40" class="grid">
      <div
        *cdkVirtualFor="let inst of instruments$ | async; trackBy: trackById"
        class="row">
        {{ inst.symbol }} — {{ inst.name }}
      </div>
    </cdk-virtual-scroll-viewport>
  `,
  styles: [`.grid { height: 480px; } .row { height: 40px; }`],
})
export class InstrumentTableComponent {
  // 10,000+ rows, but only ~12 ever exist in the DOM at once.
  readonly instruments$ = inject(InstrumentService).all$;

  // trackBy stops Angular re-creating recycled rows on every scroll frame.
  trackById = (_: number, inst: Instrument) => inst.id;
}

Почему это работает. Без виртуализации браузер строит, размещает и отрисовывает 10 000 DOM-узлов, а Angular прогоняет обнаружение изменений по всем им — O(n) на каждом цикле. Viewport из CDK рендерит только строки в видимом окне (плюс небольшой буфер) и сдвигает spacer-элемент, чтобы скроллбар оставался корректным, поэтому размер DOM и стоимость CD остаются O(видимых) ≈ константа независимо от размера набора данных.

5. Сервис WebSocket с переподключением

Задача. «Напишите типобезопасный Angular-сервис, оборачивающий WebSocket, который доставляет живые котировки. Он должен автоматически переподключаться с экспоненциальной задержкой при разрыве соединения, предоставлять типизированный поток котировок и корректно освобождать ресурсы».

RxJS даёт почти всё это из коробки: webSocket() из rxjs/webSocket для типизированного subject и retry({ delay }) для переподключения. Вычисляйте задержку как base * 2^attempt с ограничением сверху.

Добавьте джиттер к задержке, чтобы множество клиентов не переподключались синхронно после сбоя, и сбрасывайте счётчик попыток, как только успешно приходит сообщение.

import { Injectable, inject, DestroyRef } from '@angular/core';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
import { webSocket, WebSocketSubject } from 'rxjs/webSocket';
import { Observable, timer } from 'rxjs';
import { retry, tap } from 'rxjs/operators';

interface QuoteMessage {
  symbol: string;
  bid: number;
  ask: number;
  ts: number;
}

@Injectable({ providedIn: 'root' })
export class QuoteSocketService {
  private readonly url = 'wss://feed.example.com/quotes';
  private readonly maxDelay = 30_000; // cap backoff at 30s
  private socket?: WebSocketSubject<QuoteMessage>;
  private attempt = 0;
  private destroyRef = inject(DestroyRef);

  /** Type-safe, hot, auto-reconnecting stream of quotes. */
  connect(): Observable<QuoteMessage> {
    // webSocket() lazily opens the connection on first subscribe and
    // reopens it on re-subscribe (which retry() triggers).
    this.socket = webSocket<QuoteMessage>({
      url: this.url,
      openObserver: { next: () => (this.attempt = 0) }, // reset backoff on connect
    });

    return this.socket.pipe(
      // A successful message means the connection is healthy → reset attempts.
      tap(() => (this.attempt = 0)),
      // On error (drop/close), wait with exponential backoff + jitter, then retry.
      retry({
        delay: (err: unknown) => {
          const exp = Math.min(this.maxDelay, 1000 * 2 ** this.attempt++);
          const jitter = Math.random() * 300; // de-sync reconnect storms
          console.warn(`WS dropped, retrying in ${exp + jitter}ms`, err);
          return timer(exp + jitter);
        },
      }),
      // Tear the socket down when the injecting context is destroyed.
      takeUntilDestroyed(this.destroyRef),
    );
  }

  /** Send a subscribe frame for a given symbol. */
  subscribeSymbol(symbol: string): void {
    this.socket?.next({ symbol, bid: 0, ask: 0, ts: Date.now() });
  }

  /** Explicit, graceful shutdown. */
  disconnect(): void {
    this.socket?.complete();
  }
}

Частый баг: переподписка внутри retry без сброса счётчика попыток при успешном открытии — тогда задержка растёт бесконечно даже на здоровом соединении.

6. Кастомный пайп formatPrice

Задача. «Напишите standalone-пайп formatPrice, чистый, который форматирует число как валютную строку с настраиваемым кодом валюты и числом знаков после запятой — например, 1234.5 | formatPrice:'USD':2 → $1,234.50».

Не пишите разделители разрядов вручную — используйте Intl.NumberFormat со style: 'currency'. Он сам обрабатывает локаль, группировку и символ валюты.

Оставьте его pure: true (по умолчанию), чтобы он пересчитывался только при изменении входных данных. Защититесь от null/NaN, чтобы в шаблоне никогда не появлялось «NaN».

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'formatPrice',
  standalone: true,
  pure: true, // recompute only when value/currency/decimals change
})
export class FormatPricePipe implements PipeTransform {
  transform(
    value: number | null | undefined,
    currency = 'USD',
    decimals = 2,
    locale = 'en-US',
  ): string {
    // Defensive guard — never render "NaN"/"$NaN" to users.
    if (value == null || Number.isNaN(value)) return '—';

    // Intl handles grouping, symbol placement and locale rules natively.
    return new Intl.NumberFormat(locale, {
      style: 'currency',
      currency,
      minimumFractionDigits: decimals,
      maximumFractionDigits: decimals,
    }).format(value);
  }
}

// Template:
// {{ order.price | formatPrice:'EUR':4 }}  →  "€1,234.5000"

Замечание о производительности: создание Intl.NumberFormat относительно дорого. В горячей таблице можно мемоизировать форматтеры в Map по ключу `${locale}|${currency}|${decimals}` и переиспользовать их между вызовами.

7. Асинхронный валидатор уникальности email

Задача. «Напишите асинхронный валидатор для реактивных форм, который проверяет, что email ещё не зарегистрирован, обращаясь к нашему API. Он должен делать debounce, чтобы мы не били по серверу на каждое нажатие клавиши, и работать в режиме fail open (не блокировать форму), если API отдаёт ошибку».

AsyncValidatorFn возвращает Observable, который эмитит один раз и завершается. Используйте timer(debounce) + switchMap, чтобы каждое нажатие отменяло предыдущий незавершённый запрос.

Сопоставьте результат «занято» с объектом ошибки валидации { emailTaken: true }, а null — с валидным значением. Используйте catchError, чтобы вернуть null (валидно), и сбой API никогда не запирал пользователя.

import { AbstractControl, AsyncValidatorFn, ValidationErrors } from '@angular/forms';
import { Observable, of, timer } from 'rxjs';
import { switchMap, map, catchError, first } from 'rxjs/operators';

interface UserApi {
  isEmailTaken(email: string): Observable<boolean>;
}

/** Factory so the validator can be configured with a service + debounce. */
export function uniqueEmailValidator(
  api: UserApi,
  debounceMs = 400,
): AsyncValidatorFn {
  return (control: AbstractControl): Observable<ValidationErrors | null> => {
    const email = (control.value ?? '').trim();
    if (!email) return of(null); // empty → let required handle it

    // timer() debounces; switchMap cancels the previous request on new input.
    return timer(debounceMs).pipe(
      switchMap(() => api.isEmailTaken(email)),
      map(taken => (taken ? { emailTaken: true } : null)),
      // Fail open: a network/API error must NOT block form submission.
      catchError(() => of(null)),
      first(), // emit once & complete — required for async validators
    );
  };
}

// Wiring in a component:
// email: new FormControl('', {
//   validators: [Validators.required, Validators.email],
//   asyncValidators: [uniqueEmailValidator(this.userApi)],
//   updateOn: 'blur', // optional: validate on blur instead of every keystroke
// });

Подвох: асинхронные валидаторы обязаны завершаться. Если ваш источник — долгоживущий поток, завершите его через first()/take(1), иначе контрол навсегда застрянет в статусе PENDING.

8. Стор на BehaviorSubject

Задача. «Без NgRx постройте минимальный реактивный сервис состояния для списка заказов: приватное хранилище состояния, публичный observable state$ только для чтения, select() для срезов и update(), который иммутабельно патчит состояние».

private BehaviorSubject хранит текущее состояние и реплеит его новым подписчикам. Открывайте наружу только .asObservable(), чтобы вызывающий код не мог пушить в него напрямую.

select() = state$.pipe(map(selector), distinctUntilChanged()), чтобы подписчиков уведомляли только когда их срез действительно меняется.

import { Injectable } from '@angular/core';
import { BehaviorSubject, Observable } from 'rxjs';
import { map, distinctUntilChanged } from 'rxjs/operators';

interface OrdersState {
  orders: Order[];
  loading: boolean;
  error: string | null;
}

const initialState: OrdersState = { orders: [], loading: false, error: null };

@Injectable({ providedIn: 'root' })
export class OrdersStore {
  // Private — the ONLY thing allowed to call .next().
  private readonly subject = new BehaviorSubject<OrdersState>(initialState);

  // Public, read-only stream. Replays current state to late subscribers.
  readonly state$: Observable<OrdersState> = this.subject.asObservable();

  /** Synchronous snapshot for imperative reads. */
  private get state(): OrdersState {
    return this.subject.value;
  }

  /** Subscribe to a derived slice; emits only when that slice changes. */
  select<T>(selector: (s: OrdersState) => T): Observable<T> {
    return this.state$.pipe(map(selector), distinctUntilChanged());
  }

  /** Immutably patch state (accepts a partial or an updater function). */
  update(patch: Partial<OrdersState> | ((s: OrdersState) => Partial<OrdersState>)): void {
    const delta = typeof patch === 'function' ? patch(this.state) : patch;
    this.subject.next({ ...this.state, ...delta }); // new object → new reference
  }

  // --- Example domain actions built on update() ---
  addOrder(order: Order): void {
    this.update(s => ({ orders: [...s.orders, order] })); // never mutate the array
  }
  setLoading(loading: boolean): void {
    this.update({ loading });
  }
}

// Usage in a component:
// orders$ = this.store.select(s => s.orders);
// buys$   = this.store.select(s => s.orders.filter(o => o.side === 'BUY'));

Современная альтернатива: та же форма на сигналах — signal<OrdersState>() для состояния и computed() для селекторов — даёт синхронное чтение и тонкую реактивность без подписок. Та же архитектура, меньше шаблонного кода.

9. Выпрямите вложенный массив

Задача. «Выпрямите массив произвольной вложенности в один плоский массив. Покажите рекурсивное решение, итеративное и однострочное на встроенном методе».

Для каждого элемента: если это массив — рекурсивно разворачиваем его; иначе добавляем значение. Рекурсивная версия самая естественная; итеративная меняет стек вызовов на явный стек, чтобы избежать переполнения при глубокой рекурсии.

type Nested<T> = (T | Nested<T>)[];

// 1) Recursive — clean, but can overflow the stack on very deep input.
function flattenRecursive<T>(arr: Nested<T>): T[] {
  return arr.reduce<T[]>((acc, item) => {
    return Array.isArray(item)
      ? acc.concat(flattenRecursive(item)) // dive into nested array
      : (acc.push(item), acc);             // leaf value
  }, []);
}

// 2) Iterative with an explicit stack — no recursion-depth limit.
function flattenIterative<T>(arr: Nested<T>): T[] {
  const stack: (T | Nested<T>)[] = [...arr];
  const result: T[] = [];
  while (stack.length) {
    const next = stack.pop()!;            // take from the end (O(1))
    if (Array.isArray(next)) {
      stack.push(...next);                // push children back on
    } else {
      result.push(next);
    }
  }
  return result.reverse();                // we popped from the end → restore order
}

// 3) Built-in — flat(Infinity) flattens to any depth.
const flat = ([1, [2, [3, [4]]]] as Nested<number>).flat(Infinity); // [1,2,3,4]

Сложность по времени. O(n), где n — общее число элементов на всех уровнях. Рекурсивная версия использует O(d) памяти стека вызовов (d = глубина вложенности); итеративная использует явный стек на O(n), но не переполняет стек вызовов.

10. Глубокое клонирование (без structuredClone)

Задача. «Реализуйте глубокое клонирование без structuredClone и без JSON.parse(JSON.stringify()). Обработайте вложенные объекты, массивы, Date и null».

Рекурсия. Обработайте особые случаи первыми — null, примитивы, затем Date и массивы — прежде чем переходить к общему случаю объекта. typeof отличает примитивы от объектов.

Дополнительная надёжность: отслеживайте посещённые объекты в WeakMap, чтобы циклические ссылки не приводили к бесконечной рекурсии.

function deepClone<T>(value: T, seen = new WeakMap<object, unknown>()): T {
  // 1) Primitives and null/undefined are copied by value — return as-is.
  if (value === null || typeof value !== 'object') {
    return value;
  }

  // 2) Date — recreate from its timestamp.
  if (value instanceof Date) {
    return new Date(value.getTime()) as unknown as T;
  }

  // 3) Circular reference guard — return the already-cloned copy.
  if (seen.has(value as object)) {
    return seen.get(value as object) as T;
  }

  // 4) Array — clone each element recursively.
  if (Array.isArray(value)) {
    const arr: unknown[] = [];
    seen.set(value as object, arr);
    for (const item of value) arr.push(deepClone(item, seen));
    return arr as unknown as T;
  }

  // 5) Plain object — clone each own enumerable property recursively.
  const out: Record<string, unknown> = {};
  seen.set(value as object, out);
  for (const key of Object.keys(value as object)) {
    out[key] = deepClone((value as Record<string, unknown>)[key], seen);
  }
  return out as T;
}

// Example:
const original = { id: 1, when: new Date(), tags: ['a', ['b']], meta: { ok: true } };
const copy = deepClone(original);
copy.meta.ok = false;          // original.meta.ok stays true ✅

Сложность по времени. O(n) по общему числу узлов (ключи + элементы массивов). Память O(n) на клон плюс O(глубины) на рекурсию.

Почему не JSON.parse(JSON.stringify(x))? Он молча теряет undefined, функции и Symbol, превращает Date в строку, калечит NaN/Infinity в null и падает на циклических ссылках.

11. Найдите дубликаты по ключу

Задача. «Дан массив заказов; найдите те, у кого symbol встречается более одного раза. Сделайте это за один проход».

Подсчитайте вхождения каждого ключа в Map за один проход, затем соберите элементы, у которых счётчик ключа > 1. Set «увиденных» ключей тоже подойдёт, если нужны только ключи.

// Return the items that share a duplicated key value.
function findDuplicatesBy<T, K>(items: T[], keyFn: (item: T) => K): T[] {
  const counts = new Map<K, number>();
  // Pass 1: tally how many times each key appears — O(n).
  for (const item of items) {
    const k = keyFn(item);
    counts.set(k, (counts.get(k) ?? 0) + 1);
  }
  // Pass 2: keep items whose key occurred more than once — O(n).
  return items.filter(item => (counts.get(keyFn(item)) ?? 0) > 1);
}

// Just the duplicated KEY values, using a Set:
function duplicateKeys<T, K>(items: T[], keyFn: (item: T) => K): K[] {
  const seen = new Set<K>();
  const dupes = new Set<K>();
  for (const item of items) {
    const k = keyFn(item);
    if (seen.has(k)) dupes.add(k); // second+ time we've seen it
    else seen.add(k);
  }
  return [...dupes];
}

// Usage:
const dups = findDuplicatesBy(orders, o => o.symbol);
const dupSymbols = duplicateKeys(orders, o => o.symbol); // ['EURUSD', ...]

Сложность по времени. O(n) с O(k) дополнительной памяти (k = число различных ключей). Поиск в Map/Set в среднем O(1) — намного лучше наивного O(n²) с вложенным циклом.

12. Сгруппируйте массив по ключу

Задача. «Сгруппируйте Order[] по symbol в Record<string, Order[]> с помощью reduce».

Сворачивайте в объект-аккумулятор. Для каждого элемента лениво создавайте массив- корзину, если её ещё нет (acc[key] ??= []), затем добавляйте в неё.

// Generic group-by → Record<key, items[]>
function groupBy<T, K extends PropertyKey>(
  items: T[],
  keyFn: (item: T) => K,
): Record<K, T[]> {
  return items.reduce((acc, item) => {
    const key = keyFn(item);
    // Create the bucket on first sight, then append.
    (acc[key] ??= []).push(item);
    return acc;
  }, {} as Record<K, T[]>);
}

// Orders grouped by symbol:
const orders: Order[] = [
  { id: 1, symbol: 'EURUSD', side: 'BUY',  quantity: 100, price: 1.07, status: 'FILLED' },
  { id: 2, symbol: 'BTCUSD', side: 'SELL', quantity: 0.5, price: 64000, status: 'NEW' },
  { id: 3, symbol: 'EURUSD', side: 'SELL', quantity: 200, price: 1.08, status: 'NEW' },
];

const bySymbol = groupBy(orders, o => o.symbol);
// {
//   EURUSD: [ {id:1,...}, {id:3,...} ],
//   BTCUSD: [ {id:2,...} ],
// }

// Modern built-in (ES2024 / TS 5.4+): Object.groupBy(orders, o => o.symbol)

Сложность по времени. O(n) — один проход, в среднем O(1) на добавление элемента. Память O(n) на корзины.

Нативный вариант: Object.groupBy() и Map.groupBy() теперь стандартны (ES2024). Используйте их, когда целевой рантайм их поддерживает; версия на reduce — переносимый и безопасный для собеседования ответ.

13. Реализуйте Observable с нуля

Задача. «Постройте минимальный класс Observable с методом subscribe и наблюдателем из next/error/complete, чтобы показать, что вы понимаете механизм, на котором построен RxJS».

Observable — это просто функция («продюсер»), которая принимает наблюдателя и пушит в него значения. subscribe вызывает эту функцию и возвращает teardown, чтобы потребитель мог перестать получать значения.

Соблюдайте контракт: как только сработал error или complete, ни один next больше не должен доставляться. Оберните наблюдателя, чтобы это гарантировать, и вызовите teardown.

interface Observer<T> {
  next: (value: T) => void;
  error?: (err: unknown) => void;
  complete?: () => void;
}
type Teardown = () => void;
// The producer receives an observer and returns optional cleanup.
type SubscribeFn<T> = (observer: Observer<T>) => Teardown | void;

class Observable<T> {
  constructor(private readonly producer: SubscribeFn<T>) {}

  subscribe(observer: Observer<T>): { unsubscribe: Teardown } {
    let closed = false;
    let teardown: Teardown | void;

    // Guard the observer so the contract (no next after stop) holds.
    const safe: Observer<T> = {
      next: (v) => { if (!closed) observer.next(v); },
      error: (e) => {
        if (closed) return;
        closed = true;
        observer.error?.(e);
        if (teardown) teardown();      // auto-cleanup on error
      },
      complete: () => {
        if (closed) return;
        closed = true;
        observer.complete?.();
        if (teardown) teardown();      // auto-cleanup on complete
      },
    };

    // Run the producer — this is where values start flowing.
    teardown = this.producer(safe);

    return {
      unsubscribe: () => {
        if (closed) return;
        closed = true;
        if (teardown) teardown();      // stop the producer
      },
    };
  }
}

// Demo: an interval that emits 0,1,2... and cleans up its timer on unsubscribe.
const ticks$ = new Observable<number>((obs) => {
  let n = 0;
  const id = setInterval(() => obs.next(n++), 1000);
  return () => clearInterval(id);      // teardown
});

const sub = ticks$.subscribe({ next: (v) => console.log(v) });
// later: sub.unsubscribe(); // timer cleared, no leak

Ключевая идея: это ленивый и unicast — ничего не выполняется до subscribe, и каждый подписчик получает собственный запуск продюсера. Именно так работают холодные observable в RxJS; multicast им придаёт Subject.

14. Auth-интерцептор с обновлением по 401

Задача. «Напишите функциональный HTTP-интерцептор, который добавляет access-токен к запросам. На 401 он должен вызвать endpoint обновления, а затем повторить исходный запрос с новым токеном. Если несколько запросов получают 401 одновременно, обновлять токен нужно только один раз, а остальные ставить в очередь за ним».

Клонируйте запрос, чтобы добавить заголовок Authorization (запросы иммутабельны). Используйте catchError для обнаружения 401, затем switchMap в вызов обновления и повторите клонированный запрос со свежим токеном.

Для одновременных 401 разделяйте одно обновление в полёте через BehaviorSubject<string | null>. Пока идёт обновление, остальные запросы filter-уют до появления ненулевого токена, затем take(1) и продолжают — так обновление срабатывает один раз.

import { HttpInterceptorFn, HttpRequest, HttpHandlerFn, HttpErrorResponse } from '@angular/common/http';
import { inject } from '@angular/core';
import { BehaviorSubject, throwError, Observable } from 'rxjs';
import { catchError, switchMap, filter, take, finalize } from 'rxjs/operators';

// Module-level state shared across all requests = single refresh in flight.
let isRefreshing = false;
// null while a refresh is in progress; emits the new token when ready.
const refreshedToken$ = new BehaviorSubject<string | null>(null);

export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const auth = inject(AuthService);

  // Attach the current access token (clone — requests are immutable).
  const withAuth = addToken(req, auth.accessToken);

  return next(withAuth).pipe(
    catchError((err: unknown) => {
      // Only handle 401s; let everything else bubble up.
      if (err instanceof HttpErrorResponse && err.status === 401) {
        return handle401(req, next, auth);
      }
      return throwError(() => err);
    }),
  );
};

function addToken(req: HttpRequest<unknown>, token: string | null): HttpRequest<unknown> {
  return token
    ? req.clone({ setHeaders: { Authorization: `Bearer ${token}` } })
    : req;
}

function handle401(
  req: HttpRequest<unknown>,
  next: HttpHandlerFn,
  auth: AuthService,
): Observable<unknown> {
  if (!isRefreshing) {
    // First 401 in the burst kicks off THE refresh.
    isRefreshing = true;
    refreshedToken$.next(null); // signal "refresh in progress"

    return auth.refreshToken().pipe(
      switchMap((newToken: string) => {
        refreshedToken$.next(newToken);          // release the queue
        return next(addToken(req, newToken));    // retry THIS request
      }),
      catchError((refreshErr) => {
        auth.logout();                           // refresh failed → sign out
        return throwError(() => refreshErr);
      }),
      finalize(() => (isRefreshing = false)),    // reset the gate
    );
  }

  // Concurrent 401s: wait for the single refresh to publish a token, then retry.
  return refreshedToken$.pipe(
    filter((token): token is string => token !== null), // until refresh done
    take(1),                                             // one token, then proceed
    switchMap((token) => next(addToken(req, token))),    // retry queued request
  );
}

Граничные случаи, которые стоит упомянуть: не добавляйте токен к самому вызову обновления (чтобы избежать бесконечного цикла 401); пропускайте авторизацию для публичных/логин-эндпоинтов; и предусмотрите жёсткий лимит, чтобы постоянно падающее обновление не повторялось вечно.
